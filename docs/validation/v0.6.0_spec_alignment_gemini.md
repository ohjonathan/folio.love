## Notes
The spec author faced conflicting constraints: the Proposal Doc (§9.1) requires modifying `folio/enrich.py` to enforce the pending-queue cap, but the Manifest's `scope.forbidden_paths` strictly forbids touching `folio/enrich.py`. The spec correctly honored the Manifest's scope lock, but this results in an alignment blocker against the Proposal Doc's explicit commitments. Additionally, the Review Rendering Contract (§8), trust-gated surfacing (§11 / FR-811), and field renaming (§5) were dropped for scope reduction, violating the agreed-upon minimum contract in FR-813. The deliverable must be re-scoped: either the Manifest must be updated to allow modifications to `folio/enrich.py` and `folio/cli.py` to fulfill the Proposal, or the Proposal Doc must be amended to formally defer these requirements into a separate slice.
