## Minor notes
- Concurrency: no blocker found. CLI confirm/reject/confirm-doc/reject-doc and enrich acquire the library lock; unlocked review/doctor reads may observe old vs new atomic file state, but I did not find a partial-write or read-modify-write race.
- Missing-producer counter is per entry: direct-run with 5 accepted records lacking `producer` returned `missing=5`.
- `rate: null` is fine: warmup uses `None`, and non-warmup rate comes from integer counts with `total > 0`, so no `NaN` path found.
- CHANGELOG migration guidance for `graph doctor --json` is adequate.
- D.4 `_aggregate_producer_acceptance_rates` signature deviation does not require a spec-update pass; it is internal plumbing for already-specified JSON/text output. The authority citation is thin but not blocker-class.
- Coverage gap: no explicit test for multiple rejected entries under one producer with the same suppression key; current set-based implementation appears correct.
