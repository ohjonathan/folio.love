### PR-203 (should-fix) ‚Äî ¬ß10.4 self-heal sequencing differs from spec

**Spec ¬ß10.4 (line 1003):**
> **Orphan: file exists, registry row absent (SF-4 self-heal)** ‚Äî Before LLM call, build `RegistryEntry` from existing-file frontmatter and call `_register_digest` to upsert. Then proceed with rerun. Eliminates the orphan state.

**Implementation:** `generate_daily_digest` (digest.py:890) and `generate_weekly_digest` (digest.py:1115) call `_register_digest` AFTER the LLM/write succeeds (digest.py:1075 and 1282). The pre-LLM upsert sequence in ¬ß10.4 is not present.

**Functional impact:** in the orphan case, a successful rerun heals the registry via the trailing `_register_digest` call. **However**, if the LLM fails between observe-orphan and write, the orphan remains. The spec's intent is to make the heal independent of LLM success.

**Spec ¬ß10.4 stale-row warning** (line 1004):
> **Stale: registry row exists, file absent** ‚Äî Treat as fresh write (`version: 1`); registry will be overwritten by the new entry. Stderr warning (`‚Ü∑ Registry referenced missing file at {path}; regenerating.`).

**Implementation:** the fresh-write path is taken (digest.py:998-1001), but no `‚Ü∑ Registry referenced missing file ...` warning is emitted. Code path: `_load_existing_digest` returns `None` for missing file ‚Üí `path.exists()` is False ‚Üí `else` branch (line 998) silently treats as fresh write without checking for a stale registry row.

**Suggested fix:**

1. After `_compute_digest_path`, add a check for the orphan case:
   ```python
   registry_data = registry.load_registry(library_root / "registry.json")
   row_present = digest_id in registry_data.get("decks", {})
   file_present = path.exists()
   if file_present and not row_present and existing_fm is not None:
       _register_digest(config, digest_id, path, existing_fm,
                        scope_client=client, scope_engagement=engagement)
   elif row_present and not file_present:
       logger.warning("‚Ü∑ Registry referenced missing file at %s; regenerating.", path)
   ```
2. Then `generate_daily_digest` should add the corresponding tests DG-19c (stale warning) and DG-20c (orphan pre-LLM heal) per PR-201.

### PR-206 (informational) ‚Äî SF-101 capture-before-write trace verified

Spec ¬ß10.2 mandates that `tmp_path = Path(tmp.name)` MUST come AFTER `NamedTemporaryFile` creation and BEFORE any write call, so the cleanup branch can reach the orphan tempfile if `tmp.write()` fails. Implementation at `digest.py:382-404`:

```
line 387:  tmp = tempfile.NamedTemporaryFile(..., delete=False, suffix=".tmp")
line 394:  tmp_path = Path(tmp.name)            ‚Üê captured BEFORE write
line 396:  tmp.write(content)                   ‚Üê write happens here
line 399:  os.replace(tmp_path, path)
line 400:  tmp_path = None  # ownership transferred
```
