## Rationale

The spec correctly and precisely implements the narrow scope defined in Tier 4 Discovery Proposal Layer Spec §15.2. 

1. **Scope Narrowing:** It correctly limits itself to the field rename and value migration, deferring the emission of the new states (`suppressed`, `stale`, `superseded`, `accepted`) to future slices as mandated.
2. **Scope Expansion Justification:** The spec touches `folio/graph.py` and `folio/cli.py`, which were omitted from the parent spec's §15.2 scope-lock. The spec correctly identifies that failing to update these raw-dict readers would break existing diagnostics, violating the "no behavior change" rule. The expansion is minimal, well-reasoned, and approved.
3. **Carry-forward Disposition:** §8 correctly accounts for items deferred from Slice 1 and defers the appropriate new state writers to Slices 3, 4, and 6+.
4. **Behavior Preservation:** The read-fallback strategy (`from_dict` and raw readers) paired with a forward-only write strategy (`to_dict`) successfully achieves the migration without altering isolated behavior.
5. **Vocabulary Alignment:** The six new lifecycle states introduced match the §6 vocabulary exactly.
