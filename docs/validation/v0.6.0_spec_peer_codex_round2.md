- v1.1 change summary: §2 now states the return-type migration is in scope and lists seven production call sites; §7 adds a migration inventory and regression coverage notes.

## BB-2 closure (CLI contract)
- v1.1 change summary: §4.4 now makes `folio/cli.py` a MODIFY file, specifies `links review` suppression and revival rendering, chooses CLI-side rendering for producer acceptance rates while leaving `graph_doctor` as the findings provider, and defines a `graph doctor --json` object with `findings`, `producer_acceptance_rates`, and `producer_acceptance_rates_data_integrity`.
- Closure verdict: partial
- Evidence: The rendering and report-responsibility decision are now mostly implementable. However, current `graph doctor --json` emits a top-level JSON list (`json.dumps(findings)` in `folio/cli.py:1758-1759`), while v1.1 specifies a top-level object and still calls the change additive / existing-consumer-safe. That is a schema migration, not an additive key on the current shape. The spec needs to either preserve the top-level list somehow, explicitly declare the breaking JSON shape change, or define a versioned compatibility contract.
