## 3. Diagram completeness attack
No diagrams are present in the v1.1 narrowing spec, so there is no prose/diagram mismatch. The relevant completeness risks are prose/validation mismatches: the minimal schema gate says "target exists" but not "target is valid", and the manifest names runtime tests that are not present in the current tree.

## 4. Edge case inventory
- Proposal target ID exists in `registry.json`, but the target document is archived, superseded, deprecated, or otherwise not reviewable.
- Proposal target exists and relation is supported, but target type is wrong for the relation (`draws_from` to a non-evidence target).
- Both `target_exists` and `supported_relation` fail; the short-circuit shape reports only the first rule.
- Future gate wants to report multiple failures, but consumers/tests assert exactly `{"status", "rule"}`.
- `flagged_inputs=None`, `[]`, `["source"]`, `["target"]`, `["both"]`, `["inputs"]`.
- Finding with `proposal_id` missing, `None`, or empty string in stdout rendering.
- Key-only pending-proposal fixture passes even if `subject_id`, `schema_gate_result`, `input_fingerprint`, or `trust_status` values are wrong.
- Sub-slice 2 ships synthesize without promoting `_derive_trust_status`.

## 5. Security surface
No authN, authZ, command injection, path injection, secret exposure, deserialization, SSRF, or race-condition security issue surfaced in this spec round. The adversarial risk is data-integrity and operator-safety: a proposal can be displayed as schema-clean even when the target is not a valid review target.

## 6. Blind-spot identification
The author is treating "today-computable" as "available from `all_ids` without another read." That closes the missing-target hole, but it creates a new blind spot around target validity. If a target is present but semantically non-reviewable, `schema_gate_result: None` is misleading.

The second blind spot is treating a future acceptance criterion as enforcement. A v0.7.1 spec can tell v0.8.0 what to do, but unless the next manifest carries the criterion, the framework has no mechanical memory.

## 7. Risk-assessment override
I agree with the raised `medium` risk. The residual schema-gate ambiguity and JSON/stdout migration make this too risky for `low`, but the blast radius is still bounded to `folio graph doctor` and manifest/test coverage.
