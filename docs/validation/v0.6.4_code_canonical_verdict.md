## Consolidated Blockers

### DC-1 (`folio links status` missing `--include-flagged`)
*Sources:* D-AL-002 (alignment), D-PR-001 (product), D-ADV-SF-001 (adversarial, graded should-fix),
D-P-SF-001 (peer, graded should-fix). Elevated to blocker here because three
reviewers independently hit it and the contract miss is unambiguous.

Spec v1.1 §5.1 enumerates `links status` among the six commands that MUST
accept `--include-flagged`. §5.5 emits a disclosure pointing operators at
`folio links status --include-flagged`. Implementation:
- `relationship_status_summary()` (`folio/links.py:314-318`) lacks the
  `include_flagged: bool = False` parameter per spec §4.5.
- `links_status_cmd()` (`folio/cli.py:1661-1669`) lacks the
  `@click.option("--include-flagged")` decorator.
- `links_status_cmd()` empty-disclosure text points to
  `folio links review --include-flagged` instead of `folio links status --include-flagged`.

**Resolution:** Add the flag + parameter. When `include_flagged=True`,
`relationship_status_summary` partitions proposals as reviewable (not hidden
by the flag) rather than flagged-excluded. Rows retain the same shape; the
`Flagged Excluded` column becomes zero when the flag is active because the
flagged-input proposals have moved into `pending`. Update the empty-disclosure
text to match spec §5.5.

### DC-2 (`confirm-doc` / `reject-doc` silent drop in mixed clean+flagged case)
*Sources:* D-ADV-001 (adversarial, graded blocker), D-P-SF-003 (peer, graded should-fix),
with similar observations by product.

Implementation discloses flagged exclusions only when `acted == 0`. When a
document has both clean and flagged-input proposals, the clean ones are
confirmed/rejected successfully and the flagged skip is never surfaced:

```
$ folio links confirm-doc mixed_doc
✓ Confirmed 1 proposal(s)         # silently skipped 1 flagged-input proposal
```

Spec §5.6 is explicit: `When acted == 0 and flagged_excluded > 0, ...`. That
wording matches the implementation. But §11 rule 5 forbids silently incomplete
results when flagged inputs are the real cause -- mixed case is more dangerous
than all-flagged because the operator has no signal to look for skipped work.

**Resolution:**
- Code: disclose whenever `flagged_excluded > 0 and not include_flagged`,
  regardless of `acted`.
- Spec: update §5.6 to reflect the same condition.
- Tests: add mixed clean+flagged regression tests for `confirm-doc` and
  `reject-doc`.
