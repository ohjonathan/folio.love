## R1 -> R2 closure inventory

### 1. CB-2 final: flat-command-compatible option-before-scope forms

**Status:** needs-rework.

v1.2 correctly recognizes the residual issue and proposes an extended `ScopeOrCommandGroup.parse_args` in §3.3. The prose promises a value-aware walk for option-before-scope forms at spec lines 706-708, the code block implements `value_taking_long` / `value_taking_short` detection at lines 723-755, and ED-CLI-23a/b/c/d are present at lines 1185-1188.

Direct-run reproduction against a minimal Click harness copied from the v1.2 parser:

| Invocation | Result |
|---|---|
| `folio enrich --dry-run ClientA` | pass, `scope='ClientA'`, `dry_run=True` |
| `folio enrich --llm-profile X ClientA` | pass, `scope='ClientA'`, `llm_profile='X'` |
| `folio enrich --force ClientA` | pass, `scope='ClientA'`, `force=True` |
| `folio enrich diagnose ClientA` | **fails**, exit 2, `Error: No such option: --scope` |

The failure is in the proposed walk at lines 762-771: it correctly passes through the registered `diagnose` token, but then continues scanning the subcommand's positional `ClientA` and rewrites it into the group-level hidden `--scope ClientA`. Click then routes `--scope` to the `diagnose` subcommand, which does not declare that option.

Required fix: once the parser encounters a registered subcommand, it must pass the subcommand token and the remaining tokens through unchanged, or otherwise delegate the rest to Click without rewriting subcommand arguments. Keep ED-CLI-23a/b/c and make ED-CLI-23d executable against the same shape.
