**Note:** The D.3 closure spec mentions `Path.match`/`Path.glob` for §2.3 coverage. The D.4 implementation does NOT add a pathlib-path check. This is acceptable because:
1. `folio/search.py` does not currently import `pathlib` at module level (confirmed via Grep — no `from pathlib` / `import pathlib` in the module).
2. A test of `Path.match`/`Path.glob` would require tracking local-variable types through an AST walk (without a type inference engine), which is out of scope for a lightweight AST-walk guard. The D.4 fix-author's summary scope-list omits this item. The spec §2.3 omission is a well-established carry-forward the D.3 convergence recognizes as a lightweight check.
3. **Production code check**: a full-source grep for `\.match\(` / `\.glob\(` against `folio/search.py` returns no matches, so there is no latent pathlib-pattern engine in the implementation this test would catch. The risk is hypothetical-only.

The test passes against the current production code (276/276). Coverage of the 5 high-value spec §2.3 items (`regex`, `re2`, `importlib`, `compile`, `shell=True`, `globals/locals` subscripts) is landed. Mark PASS with a documented minor gap on `Path.match`/`Path.glob` that D.4 author and D.3 consolidator both accepted as below-threshold.
