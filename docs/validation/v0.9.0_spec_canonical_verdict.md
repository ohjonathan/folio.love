### CB-2 — QUERY matching is not total / not Unicode-safe

**Source:** ADV-B1-002 (codex standalone, substantive).
**Closure:** Define a module-level helper `_normalize_search_text(value: Any) -> Optional[str]` that (a) returns `None` for non-string values (silently skip, don't crash), (b) returns `None` for `None`, (c) applies `unicodedata.normalize("NFC", s).casefold()` for strings. Use it for BOTH `needle` (QUERY normalization) and all field values. Reject empty / whitespace-only QUERY at CLI and function level with a Click `callback=` validator + `ValueError` in `search()`. Fix SRC-8/SRC-9 mis-citation in §3.6 line 608 (they are producer/reason-summary tests, not Unicode tests; Unicode is SRC-12). Add new tests: SRC-34 non-string signal element, SRC-35 None field value, SRC-36 NFC/NFD equivalence, SRC-37 zero-width character no-match, SRC-38 whitespace-only QUERY rejected, SRC-39 empty QUERY rejected, SRC-40 surrogate no-crash. Add CLI tests SRC-CLI-25 empty QUERY exit-2 + SRC-CLI-26 whitespace-only QUERY exit-2.

Test SRC-CLI-13 already covers zero-matches; add SRC-CLI-27 specifically for registry-missing behavior.
