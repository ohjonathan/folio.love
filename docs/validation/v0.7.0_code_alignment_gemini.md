# Alignment Review: folio-digest-v0-7-0 (Phase D.2)

## Verdict

**Verdict: Needs Fixes**

The implementation is very close to the specification, capturing the majority of the complex trust-gating, atomic-write, and LLM-validation logic perfectly. However, it completely misses the registry state checks required by ﾂｧ10.4 for "Stale" and "Orphan" files, and misses 6 mandatory tests listed in the spec.

## Validation Findings

**Strengths:**
- **Dataclasses & Signatures (ﾂｧ3.1 & ﾂｧ3.2):** Implemented exactly as specified.
- **CLI & Lock (ﾂｧ3.3):** The `digest` group is registered correctly using `ScopeOrCommandGroup` and serialized with `library_lock`. The `refresh` command correctly patches `cli.py:1053-1062` to emit the subtype-aware guidance message.
- **Stdout Examples (ﾂｧ4):** `_emit_digest_result` handles the three-branch pluralization, `笨伝/`笨輿 prefixes, `--week --include-flagged` advisory, and `--include-flagged` echo correctly.
- **Heading Validation (ﾂｧ9.6):** `_validate_body_sections` correctly implements the B-101 ownership split. It only validates the LLM-owned headings, allowing the system to append `Trust Notes` and `Documents Drawn From` without triggering a retry.
- **Frontmatter Shape (ﾂｧ8.1):** Accurately rendered without source fields.
- **Manifest Scope (ﾂｧ2.1):** Diff confirms only allowed paths were modified.

**Gaps:**
- **Orphan/Stale Registry States (ﾂｧ10.4):** Neither `generate_daily_digest` nor `generate_weekly_digest` check `registry.json` to determine if an existing file is orphaned (file exists, row absent) or stale (row exists, file absent) *before* the LLM call.
- **Mandatory Test Coverage (ﾂｧ13.1 & ﾂｧ13.2):** 6 tests required by the spec are missing.

## Issues

### ALIGN-201: Orphan file self-heal (SF-4) not implemented
**Spec:** ﾂｧ10.4 Rerun corruption fallback - "Orphan: file exists, registry row absent (SF-4 self-heal)... Before LLM call, build RegistryEntry from existing-file frontmatter and call `_register_digest` to upsert. Then proceed with rerun."
**Implementation:** `folio/digest.py`'s `generate_daily_digest` and `generate_weekly_digest` use `_load_existing_digest(path)` to increment versions but never check `registry.json` to see if the digest row is missing. They proceed directly to the LLM call without calling `_register_digest` to self-heal the orphan state first.
**Action:** Before the LLM call, query the registry for `digest_id`. If the file exists but the row is absent, parse the frontmatter and call `_register_digest` immediately.

### ALIGN-202: Stale registry fallback not implemented
**Spec:** ﾂｧ10.4 Rerun corruption fallback - "Stale: registry row exists, file absent... Treat as fresh write (`version: 1`); registry will be overwritten by the new entry. Stderr warning (`竊ｷ Registry referenced missing file at {path}; regenerating.`)."
**Implementation:** `folio/digest.py` only checks `path.exists()`. It does not check the registry to see if a row exists when the file is absent, and fails to emit the required stderr warning.
**Action:** When `path.exists()` is False, check if the `digest_id` exists in the registry. If it does, emit the warning and treat it as a fresh write.

### ALIGN-203: Missing mandatory test cases
**Spec:** ﾂｧ13.1 and ﾂｧ13.2 enumerate exact test IDs that must be implemented.
**Implementation:** The following 6 tests are completely missing from the test suites:
- `DG-11c` (`test_collect_daily_inputs_non_dict_frontmatter_fail_open`)
- `DG-14` (`test_collect_daily_inputs_no_mtime_inference`)
- `DG-19c` (`test_generate_daily_digest_rerun_stale_registry_file_absent`)
- `DG-20c` (`test_generate_daily_digest_orphan_self_heal`)
- `DG-CLI-9d` (`test_digest_success_stdout_include_flagged_echo_plural`)
- `DG-CLI-18` (`test_digest_concurrent_invocation_serializes_or_locks`)
**Action:** Implement the missing tests in `tests/test_digest.py` and `tests/test_cli_digest.py`.

## Detailed Findings

1. `folio/digest.py`: Dataclasses `DigestFlaggedCounts`, `DailyInputSelection`, and `DigestResult` are declared exactly per spec.
2. `folio/cli.py`: The `digest` CLI group properly calls `library_lock(config.library_root, "digest")` to serialize modifications per SF-103.
3. `folio/digest.py`: `_validate_body_sections` loops strictly over the provided `llm_owned_headings`, closing the B-101 hallucination loop.
4. `folio/digest.py`: `_atomic_write` implements the SF-101 fix by capturing `tmp_path` *before* the write attempt, ensuring cleanup on a failed write.
5. Missing check in `generate_daily_digest` (around lines 558-580) and `generate_weekly_digest` (around lines 726-748): The current code structure bypasses `registry.json` entirely for its own existence checks, preventing it from resolving the corrupted states defined in ﾂｧ10.4.
