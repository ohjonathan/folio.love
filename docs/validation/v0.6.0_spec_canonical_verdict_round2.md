## Rationale

Round 2 shows significant convergence: alignment and product lenses both flipped to Approve after v1.1's §11 carry-forward table and §4.4 copy inventory. Peer and adversarial lenses surface narrow factual defects in v1.1 additions:

- **Preserved blocker B3R2-1 (claude-sub X-R2-1):** §11 carry-forward row for PR-A-14 asserts "all `SUPPORTED_RELATIONS` are asymmetric and already carry direction via source/target preserved separately in `basis_fingerprint`." This contradicts §3 mitigations row 3, which describes the fingerprint as `(sorted(source_id, target_id), relation_kind, ...)`. The fingerprint itself does NOT preserve direction — only the separate `source_id` / `target_id` fields in `_build_rejection_key`'s 4-tuple do. Spec-internal factual contradiction. Direct-citation evidence.

- **Preserved blocker B3R2-2 (codex BB-2):** §4.4 `--json` schema change from "additive" to top-level-object-replaces-top-level-list is a breaking change for existing consumers of `folio graph doctor --json`. The current code at `cli.py:1758-1759` emits `json.dumps(findings)` — a list, not an object. Spec's "additive" framing is wrong. Direct-run evidence.

- **Downgraded to should-fix (codex BB-1 partial):** `graph.py:150` is labeled as "second `graph_status`" but is actually inside `graph_doctor`. The 7-site inventory is correct and the test coverage is in place; only the wording on one row is wrong. Downgraded from blocker to should-fix-class — it does not break implementability.

- **Carried should-fix:** SF-5 (--show-suppressed flag or OQ commitment) still open from Round 1.

- **New minors:** MN-R2-1 (same as the downgraded codex BB-1 mislabel); MN-R2-2 (rate boundary test at 0.50 missing; within-bucket ordering unspecified).
