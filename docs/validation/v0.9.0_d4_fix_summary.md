### DCB-4 — SRC-23 forbidden-symbol guard weaker than spec §2.3

**Files changed:** `tests/test_search.py::test_no_forbidden_symbols_in_module` (lines 501–565).

**Change:** Extended AST walk to cover the full §2.3 spec set:
- Forbidden imports now include `regex`, `re2`, `importlib` (not just `re`, `fnmatch`).
- Forbidden calls: extended to `regex.*`, `importlib.import_module()`, builtin `compile()`, any `subprocess.*` call with `shell=True` kwarg.
- New detection: subscript access on `globals()` / `locals()` (via `ast.Subscript` with `Call(Name('globals'|'locals'))` as value).
- Substring grep fallback retained for `pickle.loads`, `eval(`, `exec(`, `os.system`, `yaml.load(`, `__import__(`.

**Regression:** Production code (`folio/search.py`) continues to pass the hardened test.

### DCB-5 — Public `search(limit=-1)` not enforced

**Files changed:** `folio/search.py::search` (lines 240–244), new `tests/test_search.py::test_search_negative_limit_raises_value_error` (SRC-43).
