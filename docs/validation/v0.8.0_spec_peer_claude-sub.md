## Summary

The spec is well-structured and, with the v0.7.1 sibling in-hand, a Phase-C Claude Opus author has a clear path. Scope lock, dataclass shape, §5 field mapping, and test coverage are largely mechanical. However, there is one BLOCKER ambiguity around the return shape of `collect_pending_relationship_proposals` that, if left unresolved, forces the implementation author to either mis-wire the excluded count or invent a helper that §2.1/§2.2 forbid (links.py is scope-locked). There are also several should-fix items around envelope/dataclass-renderer coherence (§3.1 vs §3.3), the `ScopeOrCommandGroup` vs `@cli.command()` choice, and one missing read-only-dep declaration. Fix B-1 and the B.3 meta can ship; tests already cover the critical invariants.

## Blockers

| ID | Claim | Evidence | Impact | Fix |
|---|---|---|---|---|
| B-1 | Spec misstates the return shape of `collect_pending_relationship_proposals`, breaking §5 rule-4 excluded-count derivation. | Spec §9 SYN-7 asserts the function "returns `(views, N)`" (an int). Actual signature in `folio/links.py:203-210` is `-> tuple[list[RelationshipProposalView], SuppressionCounts]`. `SuppressionCounts` is a dataclass with `rejection_memory: dict` and `flagged_input: int` (links.py:37-46). The scalar excluded count lives at `counts.flagged_input`, not the tuple's second slot as an int. Graph's consumer at `folio/graph.py:157, 224` discards counts entirely (`_`), so the synthesize author has no in-slice precedent to copy. | A Phase-C author writing `views, N = collect_pending_...(...)` then `report.excluded_flagged_count = N` will type-error at runtime (comparing a dataclass to 0, iterating incorrectly) or silently mis-populate the envelope. §5 rule 4 ("excluded count surfaced") becomes unreachable; SYN-7, SYN-CLI-5, SYN-CLI-6, E5 all drift. | Amend §1.1.4, §3.2, §5 rule 4, §6 mapping, and §9 SYN-7 to state: "call returns `(views, counts)` where `counts` is `folio.links.SuppressionCounts`; `report.excluded_flagged_count = counts.flagged_input`." Add `folio.links.SuppressionCounts` to the §2.4 read-only-deps table. |
