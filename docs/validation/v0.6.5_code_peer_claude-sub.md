## 7. Findings

### Blockers: none

### Should-fix (S)

**S-1 (none).** No must-fix spec divergences identified.

### Minor (M)

**M-1. T-11 missing (graph doctor regression).** Spec §7.1 lists `test_graph_doctor_honors_rejection_memory` as a required regression guard for the §4.3 `graph.py:246-260` call-site change. The test is not present in tests/test_graph_cli.py (or elsewhere). The behavior is correct — the code passes `apply_rejection_memory=True` explicitly and T-5 covers the underlying `suggest_person_merges` filter — but the spec-named regression guard is absent. Recommendation: add a single `test_graph_doctor_honors_rejection_memory` in tests/test_graph_cli.py that seeds a rejection, runs `folio graph doctor --json`, and asserts the `duplicate_person_candidate` finding is absent for the rejected pair. Low effort (~20 lines); mirrors T-11b's structure.

**M-2. `reasons: none` vs empty-list CLI output.** `folio/cli.py:2605` renders `reasons_str = "none"` when the `reasons_at_rejection` list is empty (preemptive-reject path). Spec §4.2 PR-SF-2 quotes the format string as `(reasons: {', '.join(reasons)})` which would produce `(reasons: )` for an empty list. Substituting `none` is a UX improvement but a literal copy divergence. Either update the spec PR-SF-2 copy to permit `none` explicitly, or change the CLI to render the empty join. Not blocking; the preemptive-reject path is itself an edge case.

**M-3. T-10 N>1 branch not explicitly asserted.** The `test_cli_suggest_merges_renders_suppression_count_pluralization_*` tests cover the zero and one branches but not `N>1`. Suggest adding a third test that seeds two rejected pairs and asserts the `"2 merge candidates suppressed…"` output. Low effort.

**M-4. T-10c sub-branches not explicitly asserted.** `test_cli_suggest_merges_disclosure_on_no_candidates` covers the empty-registry path. The `--type` non-person branch and the "empty page after `--page N`" edge are not separately asserted. Both are handled in `suggest_merges_cmd` (disclosure renders from `_render_merge_disclosure` on all paths) and behave correctly by static inspection, but the spec enumerated both in T-10c.

All M-items are non-blocking.
