## Blocker (must close before Phase C)

### B-101 — Trust Notes ownership/validation contradiction (ADV-101)

**Source:** codex adversarial ADV-101.

**Evidence (static reproduction):**
- §9.1 lists `## Trust Notes` as a required heading; `_validate_body_sections` enforces presence.
- §9.3 says Trust Notes are rendered programmatically (NOT LLM-generated).
- §9.4 corrective re-prompt for duplicates: "omit `## Trust Notes` from your output."
- §9.5 strips any LLM-emitted `## Trust Notes` before appending the programmatic block.

Reproduction: a compliant LLM that omits `## Trust Notes` (per §9.3 system ownership) triggers `_validate_body_sections` to report `## Trust Notes` as missing → §9.4 corrective retry → if second response also omits, exit non-zero per §11. Even if the second response includes Trust Notes, §9.5 strips it anyway, making the retry wasteful.

**Required fix in v1.2:** Separate **LLM-owned** from **system-owned** required headings. `_validate_body_sections` MUST validate only LLM-owned headings (Daily: Summary, What Moved Today, Emerging Risks / Open Questions, Suggested Follow-Ups; Weekly: Weekly Summary, What Changed This Week, Cross-Cutting Themes, Decisions / Risks To Track, Next Week Lookahead). System-owned sections (Documents Drawn From, Trust Notes for daily; Daily Digests Drawn From, Trust Notes for weekly) are appended/inserted by the system AFTER LLM validation passes; they are NEVER part of the LLM validation contract. Add a regression test where the first LLM response omits `## Trust Notes` and succeeds without retry.

## Should-fix (close in v1.2)

### SF-101 — `_atomic_write` tempfile capture ordering bug (PR-101)

**Source:** peer PR-101.

In v1.1 §10.2, `tmp_path = Path(tmp.name)` is INSIDE the `with` block AFTER `tmp.write(content)`. If `write()` raises (the most common failure mode the cleanup was meant to handle), `tmp_path` stays `None` and the except branch's `if tmp_path is not None` skips cleanup — exactly the orphan-tempfile bug PR-7 was trying to close. DG-20b passes vacuously unless the mock pins failure to inside `tmp.write`.

**Fix:** Capture `tmp_path` BEFORE the write call:

```python
tmp_path: Path | None = None
try:
    tmp = tempfile.NamedTemporaryFile(
        mode="w", encoding="utf-8", dir=str(path.parent), delete=False, suffix=".tmp"
    )
    tmp_path = Path(tmp.name)
    try:
        tmp.write(content)
    finally:
        tmp.close()
    os.replace(tmp_path, path)
    tmp_path = None  # ownership transferred
except Exception:
    if tmp_path is not None and tmp_path.exists():
        tmp_path.unlink(missing_ok=True)
    raise
```

Also pin DG-20b to mock `tmp.write` (not the outer flow) so the test catches the regression.

### SF-103 — Orphan self-heal not lock-protected (ADV-103)

**Source:** codex ADV-103.

§10.4 says orphan digest is self-healed by `_register_digest` before LLM call; §3.3 doesn't specify `library_lock` around the digest mutation path. Existing mutating CLI paths (`refresh`, `enrich`, `convert`) use `library_lock(library_root, "<task>")`. Without lock, two concurrent `folio digest <same-scope> --date <same-day>` runs can both observe orphan, both register, both write — last writer wins; one synthesis lost.

**Fix:** Add to §3.3 (digest CLI body) explicit `library_lock(library_root, "digest")` acquisition wrapping the entire generate/write/register sequence, including the §10.4 self-heal step. Match the `refresh` pattern. Add a §10 "Concurrency" subsection. Add a regression test (DG-CLI-18) — concurrent dispatch raises `LibraryLockError`.

### SF-104 — Heading stripping not fence-aware (ADV-104)

**Source:** codex ADV-104.

`_strip_duplicate_section` and `_validate_body_sections` use raw `## ` heading scanning. An LLM can quote source content with fenced markdown containing `## Trust Notes` inside a code fence; the raw scanner treats it as a real section, triggers retry or strips quoted content. ATX heading variants (`## Trust Notes ##`, trailing whitespace, indented) also evade exact `## ` matching.

**Fix:** Specify in §3.2 + §9.5 that heading detection MUST track fenced code blocks (` ``` ` and `~~~ `) and skip headings inside fences. Match ATX headings with optional trailing `#` and trailing whitespace. Add tests DG-23e (fenced `## Trust Notes` ignored), DG-23f (`## Trust Notes ##` matched), DG-23g (`## Trust Notes  ` with trailing whitespace matched).

### SF-105 — Forbidden symbols miss dynamic import/introspection (ADV-106)

**Source:** codex ADV-106.

§2.3 lists direct unsafe symbols but not dynamic equivalents: `__import__("os").system(...)`, `importlib.import_module("subprocess").run(..., shell=True)`, `globals()["..."](...)`, `getattr(module, "system")(...)`. Substring-only grep tests don't catch these.

**Fix:** Extend §2.3 spec-side forbidden list with `__import__(`, `importlib.import_module(`, `globals()[`, `locals()[`. Note that user-controlled `getattr`/`setattr` to module attrs is hard to lint statically; recommend AST-based test (DG-30c) for `subprocess.run(..., shell=True)` and any dynamic import with non-literal argument. Concrete: DG-30c uses `ast` to walk `folio/digest.py`.
