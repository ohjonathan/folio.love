## D.4 Fix Pass Summary

Closes both D.3 canonical blockers (DCB-1, DCB-2) and ten of the fifteen should-fix items. Five should-fix items deferred to v0.8.1 or accepted-as-is per D.3 disposition.

## Blocker closures

### DCB-1 — Document-ID scope contract implemented

- New `_resolve_scope(config, scope_arg) -> (doc_id, subtree_scope)` helper in `folio/synthesize.py`.
  - Exact `registry.decks` key match → `(doc_id, None)` → passed as `doc_id=` to `collect_pending_relationship_proposals`.
  - Subtree match on `entry.markdown_path` or `entry.deck_dir` (via `folio.links._matches_scope`) → `(None, subtree_scope)` → passed as `scope=`.
  - Library-wide sentinels (`None`, `-`, `""`, whitespace) → `(None, None)`.
- `synthesize()` now unpacks the resolver tuple and forwards both kwargs to the collector (`folio/synthesize.py:129-134`).
- New test `SYN-CLI-23` (`test_synthesize_document_id_scope_routes_via_doc_id`) asserts `folio synthesize <doc_id>` returns only findings whose `source_id == <doc_id>`.
- Existing `test_synthesize_calls_collect_function` updated to populate the registry for scope resolution and assert `kwargs["doc_id"] is None` on subtree scopes.

### DCB-2 — Invalid-scope exits 1 with stderr error

- New `ScopeResolutionError` exception raised from `_resolve_scope` when the arg matches neither a doc ID nor any subtree.
- CLI catches the exception at `folio/cli.py:1986-1988`:
  ```python
  except ScopeResolutionError as exc:
      click.echo(f"Error: {exc}", err=True)
      ctx.exit(1)
  ```
- Test `SYN-CLI-7` (renamed `test_synthesize_invalid_scope_exits_nonzero`) now asserts exit 1 (previously the test was inverted — documented the D.2 violation rather than closing it).
- New test `SYN-CLI-24` (`test_synthesize_invalid_scope_exits_1_with_stderr_error`) asserts the exact stderr format.
- Empty-library case is distinct from invalid-scope: omitted `scope` → exit 0 with CB-5 breadcrumb; explicit unresolvable scope → exit 1.
