## 2. Should-Fix Items

### S1. Eliminate double blank-detection scan

**File:** `folio/pipeline/images.py`
**Problem:** `_validate_image()` (called inside `extract()` at line 88) and `extract_with_metadata()` (line 141) each independently call `_is_mostly_blank()`, which materializes the full pixel array as a Python list via `Image.getdata()`. At 150 DPI, each 1920x1080 image produces ~2M Python int objects (~56 MB). For a 100-slide deck called via `extract_with_metadata()`, both scans run, producing ~11 GB of transient allocations.

**Fix:** Cache the blank-detection result from `_validate_image()` so `extract_with_metadata()` can reuse it without re-scanning.

**Implementation:**

1. Change `_validate_image()` to return a result dict instead of just logging:

```python
def _validate_image(image_path: Path, image: Image.Image, slide_num: int) -> dict:
    """Validate image and return metadata. Warns on suspicious images."""
    size_bytes = image_path.stat().st_size
    width, height = image.size
    is_blank = False
    is_tiny = False

    if size_bytes == 0:
        logger.warning("Slide %d: image file is empty (0 bytes)", slide_num)
        return {"is_blank": False, "is_tiny": False, "width": 0, "height": 0}

    if width < 100 or height < 100:
        logger.warning("Slide %d: unusually small (%dx%d)", slide_num, width, height)
        is_tiny = True

    if _is_mostly_blank(image, threshold=0.95):
        logger.warning("Slide %d: appears mostly blank", slide_num)
        is_blank = True

    return {"is_blank": is_blank, "is_tiny": is_tiny, "width": width, "height": height}
```

2. In `extract()`, collect validation results into a module-level cache dict keyed by resolved path string:

```python
# Module-level (cleared after each extract_with_metadata call)
_validation_cache: dict[str, dict] = {}
```

In the `extract()` loop:
```python
meta = _validate_image(image_path, image, slide_num=i)
_validation_cache[str(image_path.resolve())] = meta
```

3. In `extract_with_metadata()`, check the cache before re-scanning:

```python
for i, path in enumerate(paths, 1):
    cache_key = str(path.resolve())
    cached = _validation_cache.get(cache_key)
    if cached:
        is_blank = cached["is_blank"]
        is_tiny = cached["is_tiny"]
        width = cached["width"]
        height = cached["height"]
    else:
        with Image.open(path) as img:
            width, height = img.size
            is_blank = _is_mostly_blank(img, threshold=0.95)
            is_tiny = width < 100 or height < 100
    # ... build ImageResult
```

4. Clear the cache at the end of `extract_with_metadata()`:
```python
_validation_cache.clear()
```

**Tests:**
- Verify `_is_mostly_blank` is called once per image, not twice, when going through `extract_with_metadata()` (mock `_is_mostly_blank` and assert call count).

**Alternative (simpler, preferred if perf is acceptable):** Instead of the cache, switch `_is_mostly_blank()` to use `image.histogram()` which is a C-level operation and avoids materializing a Python list:

```python
def _is_mostly_blank(image: Image.Image, threshold: float = 0.95) -> bool:
    """Check if an image is mostly white/blank using histogram."""
    try:
        grayscale = image.convert("L")
        hist = grayscale.histogram()  # 256-bin list, C-level, ~2KB
        total = sum(hist)
        white_count = sum(hist[241:])  # pixels > 240
        return (white_count / total) > threshold
    except Exception:
        return False
```

This makes the double-scan moot (~microseconds each instead of ~50ms) and requires no architectural change. **Prefer this approach.** Still update `_validate_image` to return metadata for S1 completeness, but the perf urgency drops from "must fix" to "nice to have."

### S2. Harden HR frontmatter stripping with YAML validation

**File:** `folio/pipeline/text.py:285-289`
**Problem:** The frontmatter stripping heuristic removes everything between the first `---` and the next `---` without verifying the content is YAML. A deck whose text legitimately starts with `---\ncontent\n---` loses the first slide's content.

**Fix:** Validate the stripped content is parseable YAML before removing it.

**Implementation:**

```python
# In _parse_slide_boundaries(), replace lines 285-289 with:
text_for_hr = raw_text
if text_for_hr.startswith("---"):
    frontmatter_end = text_for_hr.find("---", 3)
    if frontmatter_end != -1:
        candidate = text_for_hr[3:frontmatter_end].strip()
        try:
            import yaml
            parsed = yaml.safe_load(candidate)
            if isinstance(parsed, dict):
                # Looks like real YAML frontmatter — strip it
                text_for_hr = text_for_hr[frontmatter_end + 3:]
        except Exception:
            pass  # Not valid YAML — keep the text intact
```

**Dependency note:** `yaml` (PyYAML) is already a dependency — imported in `converter.py:11` as `import yaml as yaml_lib`.

**File:** `folio/pipeline/images.py:182`
**Problem:** `PIL.Image.getdata()` is deprecated in Pillow and will be removed in Pillow 14 (2027-10-15). The test suite emits 9 deprecation warnings.

**Fix:** This is resolved by the histogram approach in S1 above. The replacement:

```python
def _is_mostly_blank(image: Image.Image, threshold: float = 0.95) -> bool:
    """Check if an image is mostly white/blank using histogram."""
    try:
        grayscale = image.convert("L")
        hist = grayscale.histogram()
        total = sum(hist)
        if total == 0:
            return False
        white_count = sum(hist[241:])  # pixels with value > 240
        return (white_count / total) > threshold
    except Exception:
        return False
```

### M7. Guard `_compute_timeout` against non-positive `base_timeout`

**File:** `folio/pipeline/normalize.py:145-149`

```python
def _compute_timeout(source_path: Path, base_timeout: int) -> int:
    """Scale timeout: base + 1s per MB, capped at 300s. Minimum 10s."""
    size_mb = source_path.stat().st_size / (1024 * 1024)
    scaled = max(base_timeout, 10) + int(size_mb)
    return min(scaled, 300)
```
