## 1. Problem Statement

The extraction pipeline (`normalize.py`, `images.py`, `text.py`) currently has several gaps that block Tier 1 exit criteria:

1. **No input validation** — 0-byte files, password-protected PPTX, and corrupt archives reach LibreOffice and fail with opaque errors.
2. **Blank slides produce wrong analysis** — `_validate_image()` in `images.py` logs warnings (line 84) but the caller ignores them. Blank slides still receive Claude API analysis, producing incorrect slide types and tags that contaminate frontmatter aggregation. (API token cost optimization via pre-call skip is deferred to Track B.)
3. **Fragile slide boundary detection** — `_parse_slide_boundaries()` in `text.py` (line 202) misses horizontal-rule boundaries and its numbered-section pattern (Pattern 3, line 230) is too greedy — any standalone number triggers a false boundary.
4. **No slide count reconciliation** — The mismatch between image count and text count is warned (converter.py lines 104-109) but never resolved, leading to misaligned slide text and images downstream.

### In scope (Track A)
- Input validation in `normalize.py`
- `ImageResult` dataclass and additive `extract_with_metadata()` in `images.py` (no interface break to `extract()`)

- Improved blank threshold tuning (adaptive instead of fixed 95%)

- Unicode normalization for CJK text

- Semantic content-based text↔image alignment (OCR/LLM matching)

#### 3.1.1 Add `_validate_source(source_path: Path) -> None`

New private function, called as the first line of `to_pdf()` after path resolution (after line 31).

**Checks (in order):**

| Check | How | Error |
|-------|-----|-------|
| File exists | `source_path.exists()` | `NormalizationError("Source not found: {path}")` |
| Non-zero size | `source_path.stat().st_size == 0` | `NormalizationError("Source file is empty (0 bytes): {name}")` |
| Max size 500 MB | `source_path.stat().st_size > 500 * 1024 * 1024` | `NormalizationError("Source file too large ({size_mb:.0f} MB, max 500 MB): {name}")` |
| Valid ZIP for .pptx | Open with `zipfile.is_zipfile()` | `NormalizationError("Invalid PPTX (not a ZIP archive): {name}")` |
| Not password-protected | Open as ZIP, check for `EncryptedPackage`/`EncryptedInfo` in namelist | `NormalizationError("Password-protected PPTX: {name}")` |
| Valid OOXML structure | `[Content_Types].xml` in ZIP namelist | `NormalizationError("Malformed PPTX (missing [Content_Types].xml): {name}")` |

**Validation logic (inline in `_validate_source`):**
```python
import zipfile

def _validate_source(source_path: Path) -> None:
    """Pre-flight validation before LibreOffice conversion."""
    if not source_path.exists():
        raise NormalizationError(f"Source not found: {source_path}")

    size = source_path.stat().st_size
    if size == 0:
        raise NormalizationError(
            f"Source file is empty (0 bytes): {source_path.name}"
        )
    if size > 500 * 1024 * 1024:
        size_mb = size / (1024 * 1024)
        raise NormalizationError(
            f"Source file too large ({size_mb:.0f} MB, max 500 MB): "
            f"{source_path.name}"
        )

    if source_path.suffix.lower() == ".pptx":
        if not zipfile.is_zipfile(source_path):
            raise NormalizationError(
                f"Invalid PPTX (not a ZIP archive): {source_path.name}"
            )
        try:
            with zipfile.ZipFile(source_path, "r") as zf:
                names = zf.namelist()
                if "EncryptedPackage" in names or "EncryptedInfo" in names:
                    raise NormalizationError(
                        f"Password-protected PPTX: {source_path.name}"
                    )
                if "[Content_Types].xml" not in names:
                    raise NormalizationError(
                        f"Malformed PPTX (missing [Content_Types].xml): "
                        f"{source_path.name}"
                    )
        except zipfile.BadZipFile:
            raise NormalizationError(
                f"Invalid PPTX (corrupt ZIP archive): {source_path.name}"
            )
```

> **Coverage note:** `.ppt` (legacy binary) files pass through LibreOffice conversion without OOXML validation (no ZIP structure). Encrypted PDF detection requires different heuristics (not OOXML-based) and is deferred to Track B. Current validation covers `.pptx` only; other formats rely on LibreOffice error reporting.

#### 3.2.4 Atomic temp-dir extraction (no data loss on failure)

Extract to a temp directory on the same filesystem, then atomically swap on success. This prevents `shutil.rmtree(slides_dir)` from destroying previously-good images during reconversion.

Follows existing atomic patterns: `versions.py:232-237`, `converter.py:178-181`.

```python
slides_dir = output_dir / "slides"
tmp_dir = output_dir / ".slides_tmp"
old_dir = output_dir / ".slides_old"

# Preflight: recover from interrupted previous runs
if old_dir.exists():
    if not slides_dir.exists():
        # Crashed after slides/ → .slides_old but before .slides_tmp → slides/
        old_dir.rename(slides_dir)
        logger.warning("Recovered slides/ from interrupted swap: %s", old_dir)
    else:
        # Stale leftover — slides/ is intact
        shutil.rmtree(old_dir)
        logger.debug("Cleaned up stale .slides_old: %s", old_dir)

try:
    if tmp_dir.exists():
        shutil.rmtree(tmp_dir)  # clean leftover from previous failed run
    tmp_dir.mkdir(parents=True)
    images = convert_from_path(pdf_path, dpi=dpi, fmt=fmt)

    image_paths = []
    for i, image in enumerate(images, 1):
        filename = f"slide-{i:03d}.{fmt}"
        image_path = tmp_dir / filename
        image.save(str(image_path))
        image_paths.append(image_path)

except Exception as e:
    # Failure: delete only tmp_dir, existing slides/ untouched
    if tmp_dir.exists():
        shutil.rmtree(tmp_dir)
        logger.debug("Cleaned up temp slides dir: %s", tmp_dir)
    raise ImageExtractionError(f"pdf2image failed: {e}") from e

# Success: atomic swap
if slides_dir.exists():
    slides_dir.rename(old_dir)
try:
    tmp_dir.rename(slides_dir)
except Exception:
    # Restore old slides on rename failure
    if old_dir.exists():
        old_dir.rename(slides_dir)
    raise
finally:
    if old_dir.exists():
        shutil.rmtree(old_dir)

# Rewrite paths to final location
image_paths = sorted(slides_dir.glob(f"*.{fmt}"))
return image_paths
```

> **Concurrency note:** The folio pipeline processes decks sequentially (cli.py batch loop). No concurrent access to the same `output_dir` occurs. If parallelism is added (Track B), replace fixed `.slides_tmp` naming with `tempfile.mkdtemp(dir=output_dir, prefix='.slides_tmp_')` and use file locking.

**Current state:** `SlideText` (line 15) has no empty indicator. `_parse_slide_boundaries()` (line 202) handles 3 patterns. No reconciliation.

**Data flow through text.py:**

```
Source file
    ↓
_extract_pdf(pdf_path) → dict[int, SlideText]      # per-page structured text (PDF inputs)
_extract_via_markitdown(path) → str                 # single text blob (PPTX inputs)
    ↓
_parse_slide_boundaries(raw_text) → dict[int, str]  # split blob into per-slide text
    ↓
_detect_elements(text) → list[dict]                  # classify text elements per slide
    ↓
extract_structured() → dict[int, SlideText]          # PUBLIC API: structured output
    ↓
reconcile_slide_count(slide_texts, image_count)      # align with authoritative image count
    → ReconciliationResult
```

> **Note on `_extract_pdf()`:** This function calls `_detect_elements()` internally and returns `dict[int, SlideText]` directly — it does NOT return raw strings that `extract_structured()` wraps. The PPTX path (`_extract_via_markitdown` → `_parse_slide_boundaries` → `_detect_elements`) is the only path that goes through intermediate string dicts.

**Positional alignment invariant:** Both extraction paths produce text in presentation order. Text chunk N corresponds to the Nth slide in presentation order. This invariant is maintained by PDF per-page extraction (page order = slide order) and by sequential boundary detection on PPTX text blobs. When text count ≠ image count, the positional ordering is preserved but some slides may have merged text (fewer text chunks than images) or falsely-split text (more chunks than images). Reconciliation pads/truncates to match counts and signals alignment confidence — it does not attempt content-based semantic alignment (Track B).

```python
def _split_by_hr(raw_text: str, hr_matches: list) -> dict[int, str]:
    """Split text into slides using horizontal rules as boundaries."""
    slides = {}
    slide_num = 1

    # Text before first HR
    first_text = raw_text[:hr_matches[0].start()].strip()
    if first_text:
        slides[slide_num] = first_text
        slide_num += 1

    # Text between HRs
    for i, match in enumerate(hr_matches):
        start = match.end()
        end = hr_matches[i + 1].start() if i + 1 < len(hr_matches) else len(raw_text)
        text = raw_text[start:end].strip()
        text = re.sub(r"<!--.*?-->", "", text, flags=re.DOTALL).strip()
        if text:
            slides[slide_num] = text
            slide_num += 1

    return slides
```

#### 3.3.4 New `reconcile_slide_count()` with `ReconciliationResult`

New dataclass at module level:

```python
@dataclass
class ReconciliationResult:
    """Result of reconciling text count with authoritative image count."""
    slide_texts: dict[int, SlideText]
    was_reconciled: bool
    action: str  # "none" | "padded" | "truncated"
    gaps_filled: int  # count of empty entries inserted for missing keys within range

    image_count: int
```

Updated function with gap-filling:

```python
def reconcile_slide_count(
    slide_texts: dict[int, SlideText],
    image_count: int,
) -> ReconciliationResult:
    """Pad or truncate slide text dict to match authoritative image count.

    Image count (from PDF page count) is authoritative. Text extraction
    may produce fewer slides (missed boundaries) or more (false boundaries).

    Keys are NEVER remapped — they preserve page/slide positional identity.
    Gaps within the existing key range are filled with empty SlideText entries.
    This works correctly for both extraction paths:
      - PDF path: keys are page numbers, gaps = empty pages → fill with empties
      - PPTX path: keys are already contiguous 1..N → no gaps to fill

    Returns:
        ReconciliationResult with reconciled slide_texts and metadata.
        Consumers should check alignment_confidence to assess trust.
    """
    if not slide_texts:
        # No text at all — return empty placeholders
        return ReconciliationResult(
            slide_texts={
                i: SlideText(slide_num=i, full_text="", is_empty=True)
                for i in range(1, image_count + 1)
            },
            was_reconciled=True,
            action="padded",
            gaps_filled=0,
            original_text_count=0,
            image_count=image_count,
            alignment_confidence=0.0,
        )

    original_text_count = len(slide_texts)

    # Step 1: Fill gaps within existing key range with empties
    max_existing_key = max(slide_texts.keys())
    gaps_filled = 0
    for i in range(1, max_existing_key + 1):
        if i not in slide_texts:
            slide_texts[i] = SlideText(slide_num=i, full_text="", is_empty=True)
            gaps_filled += 1

    if gaps_filled:
        logger.info(
            "Filled %d gaps in text keys (range 1..%d)",
            gaps_filled, max_existing_key,
        )

    # Step 2: Pad or truncate to match image count
    text_count = len(slide_texts)
    confidence = (
        min(original_text_count, image_count) / max(original_text_count, image_count)
        if image_count > 0 else 0.0
    )

    if text_count == image_count:
        return ReconciliationResult(
            slide_texts=dict(slide_texts),
            was_reconciled=gaps_filled > 0,
            action="none",
            gaps_filled=gaps_filled,
            original_text_count=original_text_count,
            image_count=image_count,
            alignment_confidence=confidence,
        )

    if text_count > image_count:
        logger.warning(
            "Text extraction found %d slides but only %d images. "
            "Truncating text to match image count.",
            text_count, image_count,
        )
        return ReconciliationResult(
            slide_texts={
                k: v for k, v in slide_texts.items() if k <= image_count
            },
            was_reconciled=True,
            action="truncated",
            gaps_filled=gaps_filled,
            original_text_count=original_text_count,
            image_count=image_count,
            alignment_confidence=confidence,
        )

    # text_count < image_count — pad missing slides
    logger.warning(
        "Text extraction found %d slides but %d images. "
        "Padding %d missing slides with empty text.",
        text_count, image_count, image_count - text_count,
    )
    result = dict(slide_texts)
    for i in range(1, image_count + 1):
        if i not in result:
            result[i] = SlideText(slide_num=i, full_text="", is_empty=True)
    return ReconciliationResult(
        slide_texts=result,
        was_reconciled=True,
        action="padded",
        gaps_filled=gaps_filled,
        original_text_count=original_text_count,
        image_count=image_count,
        alignment_confidence=confidence,
    )
```

```python
def _looks_like_table(text: str) -> bool:
    """Heuristic: text looks like a table if 3+ lines have consistent pipe delimiters."""
    if not text:
        return False
    lines = [l for l in text.split("\n") if l.strip()]
    pipe_lines = [l for l in lines if l.count("|") >= 2]
    return len(pipe_lines) >= 3 and len(pipe_lines) / len(lines) > 0.5
```

#### 3.3.6 PDF failure policy

`_extract_pdf()` is a **private helper** within the `extract_structured()` pipeline (see data flow above). It calls `_detect_elements()` internally and returns `dict[int, SlideText]` directly. Changes here affect the extraction step — the public `extract_structured() → dict[int, SlideText]` contract is unchanged.

**Three-layer failure policy:**

| Layer | Function | Behavior |
|-------|----------|----------|
| L1 (extraction) | `_extract_pdf()` | **Raise** `TextExtractionError` on unexpected errors. `ImportError` → raise. `pdfplumber.open()` failure → raise. `page.extract_text()` returns None → continue (legitimate empty page). |
| L2 (dispatcher) | `extract_structured()` | **Catch** `TextExtractionError`, log warning, return `{}`. This is the structured fallback boundary. |
| L3 (converter) | `convert()` | **Unchanged** — receives `{}`, continues. Log info "proceeding with image-only analysis". |

This is **structured fallback**: errors are raised at the source (L1), caught and logged at the dispatcher (L2), and the converter (L3) degrades gracefully. Not silent — every text extraction failure is logged as a warning.

In `_extract_pdf()` (text.py:172), change behavior from silent fallback to explicit errors:

- `ImportError` on pdfplumber → raise `TextExtractionError` (pdfplumber is now a required dependency)
- `pdfplumber.open()` exception → raise `TextExtractionError` (not silent empty dict)
- `page.extract_text()` returns `None` → legitimate empty page (continue, no error)

```python
def _extract_pdf(pdf_path: Path) -> dict[int, SlideText]:
    """Extract per-page structured text from PDF using pdfplumber.

    Returns dict of SlideText keyed by 1-based page number. For PDF inputs,
    page number = slide number (1:1 mapping). Empty pages are
    omitted from the result (legitimate gaps, not errors).

    Calls _detect_elements() internally — returns SlideText directly,
    NOT raw strings.

    Raises TextExtractionError on unexpected failures (L1). The caller
    (extract_structured) catches these at L2 and falls back to {}.
    """
    try:
        import pdfplumber
    except ImportError:
        raise TextExtractionError(
            "pdfplumber is required for PDF text extraction. "
            "Install with: pip install pdfplumber"
        )

    try:
        with pdfplumber.open(pdf_path) as pdf:
            slides = {}
            for i, page in enumerate(pdf.pages, 1):
                page_text = page.extract_text()
                if page_text and page_text.strip():
                    text = page_text.strip()
                    slides[i] = SlideText(
                        slide_num=i,
                        full_text=text,
                        elements=_detect_elements(text),
                    )
            return slides
    except TextExtractionError:
        raise
    except Exception as e:
        raise TextExtractionError(
            f"PDF text extraction failed for {pdf_path.name}: {e}"
        ) from e
```

> **Note on sparse keys:** When `_extract_pdf()` omits empty pages, the returned dict may have gaps (e.g., `{1: SlideText(...), 3: SlideText(...)}`). These gaps are filled with empty `SlideText` entries by `reconcile_slide_count()` (§3.3.4 Step 1) — keys are never remapped, preserving page/slide positional identity.

#### 3.4.5 Flow reconciliation metadata to frontmatter

After reconciliation, pass metadata through to frontmatter output (~5 lines in converter.py).

**`text_alignment_status` field** (derived from `alignment_confidence`):

| Condition | `text_alignment_status` | Downstream policy |
|-----------|------------------------|-------------------|
| `alignment_confidence >= 0.7` | `"accepted"` | Text-dependent outputs are trustworthy. |
| `0.3 <= alignment_confidence < 0.7` | `"degraded"` | Text-dependent outputs usable with caution. |
| `alignment_confidence < 0.3` | `"untrusted"` | Downstream consumers MUST NOT use text-dependent outputs (evidence validation, grounding summary) without manual review. |

When `text_alignment_status: "untrusted"`, enforcement is by convention — the pipeline still completes.

**Degraded-trust operational policy:**

When `text_reconciled: true` appears in frontmatter, consumers should:

| Condition | Meaning | Recommended Action |
|-----------|---------|-------------------|
| `action: "padded"` | Some slides have no extracted text. Analysis relies entirely on image content. | Generally safe. Most common cause: text boundaries missed by parser. No manual intervention unless quality issues observed. |
| `action: "truncated"` | Text extraction found more slides than images. Extra text was dropped. | Review deck for false boundary detection. If `alignment_confidence < 0.6`, flag for manual review. |
| `text_alignment_status: "untrusted"` | Severe mismatch between text and image counts. | Flag for manual text/image verification. Do not use text-dependent outputs without review. |
| `text_alignment_status: "accepted"` | Minor or no discrepancy. | Safe to proceed. Padding/truncation affects few slides. |

The default behavior is to **proceed with degraded alignment**. No automatic abort. The metadata enables downstream filtering and quality dashboards.

### 3.6 Decision Records

> **Governance note:** folio.love does not yet have a formal dependency governance policy. These ADRs serve as the initial decision records and establish the pattern. If a governance framework is adopted, link these ADRs to the corresponding policy constraints.

#### ADR-001: pdfplumber optional → required

- **Status:** Accepted
- **Approved by:** Chief Architect (pending)
- **Context:** `text._extract_pdf()` imports pdfplumber conditionally and returns an empty dict on `ImportError`. This silently degrades PDF text extraction without any signal to the user.
- **Decision:** Make pdfplumber a required dependency (`>=0.9`).
- **Consequences:** Pure Python package, ~5 MB installed. No native dependencies beyond what pdf2image/Pillow already require. Eliminates silent degradation path.

#### ADR-002: python-pptx as dev dependency

- **Status:** Accepted
- **Approved by:** Chief Architect (pending)
- **Context:** Test fixtures need to programmatically create PPTX files for validation, password-protection, and malformed-archive testing.
- **Decision:** Add python-pptx as a dev-only dependency (`>=0.6`).
- **Consequences:** ~15 MB installed, dev-only. Not required at runtime. Enables reliable, deterministic test fixture generation instead of checking in binary files.

## 8. Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| `extract_with_metadata()` re-opens images for annotation (perf) | Medium | Low | PIL `Image.open` is lazy; only reads headers for size. Blank detection reads pixels but images are already in page cache. Profile if >2s overhead on 100-slide decks. |
| Password detection false positives on valid PPTX | Low | Medium | Test against 10+ real PPTX files from the test corpus. `[Content_Types].xml` check is standard OOXML. |
| HR boundary pattern matches content HRs (not slide boundaries) | Medium | Low | Require `>=2` HRs. If false positive rate is high, gate behind PPTX-source-only. |
| `pdfplumber` promotion breaks installs on systems without C deps | Low | Medium | pdfplumber is pure Python. No native deps beyond what's already required by pdf2image/Pillow. |
| Table detection heuristic too aggressive | Medium | Low | Only affects element type annotation. No downstream logic depends on "table" vs "body" in Tier 1. |
| Atomic swap fails on cross-filesystem rename | Low | High | Same filesystem guaranteed: `tmp_dir` and `slides_dir` share `output_dir` parent. Add assertion in tests. |
| Positional alignment assumption wrong for edge-case decks | Medium | Medium | `alignment_confidence` signals mismatch severity. Empirically validate against 50-deck test corpus. True content-based semantic alignment (OCR/LLM matching) is Track B. |
| Gap-filling inserts wrong empties if PDF page numbers don't match slide order | Low | Medium | PDF page numbers = slide order is a reliable invariant for single-deck PDFs. Multi-deck concatenated PDFs are not supported. |
| Structured fallback masks persistent extraction failures | Low | Medium | L2 logs warning on every fallback. Monitoring/alerting on `text_alignment_status: "untrusted"` in frontmatter catches persistent issues. |
