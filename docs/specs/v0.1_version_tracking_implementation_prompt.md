**How change detection works:** `detect_changes()` compares slide texts by position (slide number = identity). It normalizes text via `_normalize_text()` which collapses whitespace and strips. Two slides are "unchanged" if their normalized text is identical. This is **intentionally different** from `analysis.py:_text_hash()` which hashes raw text for API input reproduction. The spec (D1) says this difference is correct — see the `_normalize_text()` docstring for the explanation.

**How versioning works:** `compute_version()` loads the previous texts cache, runs `detect_changes()`, increments the version number, persists the new history entry and texts cache, and returns `VersionInfo`. The first version marks all slides as "added". Re-conversion with identical content increments the version (audit trail) with `has_changes=False`.

**Cache format:** `.texts_cache.json` stores slide texts with string keys and a `_cache_version` marker (currently `2`). `version_history.json` stores `{"versions": [...]}` with each entry matching `VersionInfo.to_dict()`.

## Constraints

1. **Do NOT change `analysis.py` or the caching system.** Version tracking and analysis caching are independent systems with intentionally different text comparison strategies (spec D1).

2. **Do NOT change extraction stages** (`normalize.py`, `images.py`, `text.py`). These are upstream of version tracking.

3. **Do NOT change `converter.py`, `cli.py`, or `config.py`.** Version tracking is fully wired in. No pipeline changes needed.

4. **Do NOT add external dependencies.** All changes are pure Python + stdlib + existing deps.

5. **Do NOT add async/parallel operations.** Version tracking is single-process by design (spec D4 concurrency note).

6. **Do NOT build a diff engine or change visualization.** Deferred to Tier 2+.

7. **Backward compatibility:** Decks converted before version tracking (no `version_history.json` or `.texts_cache.json`) must not crash. `load_version_history()` returns `[]` and `load_texts_cache()` returns `{}` for missing files — `compute_version()` then creates v1 with all slides "added." This behavior already works; do not break it.

8. **Change detection consistency:** Version tracking uses `_normalize_text()` (whitespace-insensitive semantic comparison). Analysis caching uses `_text_hash()` (raw text hash for API input fidelity). These are different by design. A whitespace-only change correctly produces:
   - Version tracking: no new version (semantic content unchanged)
   - Analysis caching: cache miss (different API prompt bytes)

   Do NOT attempt to unify these. Both are correct for their concerns.

9. **No `_EXTRACTION_VERSION` in texts cache.** The spec (D2) explicitly says this is not needed because text diffs are self-correcting. The `save_texts_cache()` docstring (Step 2b) explains why.

10. **Write failures must crash.** `_atomic_write_json()` intentionally does not catch `OSError`. Version data is correctness-critical (spec D4). The `_atomic_write_json()` docstring (Step 2c) explains why.
