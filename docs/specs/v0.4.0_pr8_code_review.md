## Blocking Issues (3)

### B1: Anthropic `classify_error` missing `APITimeoutError`

- **File:** `folio/llm/providers.py`:82-102
- **Spec:** section 6.3
- **Found by:** R1, R2, R3 (unanimous)
- **Description:** The spec lists `APITimeoutError` as transient for Anthropic. The `classify_error` method does not check for it. It falls through to the default `return ErrorDisposition.PERMANENT`. A timeout on a vision analysis call (common under load) will not trigger retry or fallback, silently degrading to pending.
- **Fix:** Add `if isinstance(exc, anthropic.APITimeoutError): return ErrorDisposition.TRANSIENT` after the existing transient checks. Also add explicit `PermissionDeniedError` -> `PERMANENT` for spec completeness (functionally correct today via default, but fragile).

### B2: OpenAI `classify_error` missing `APITimeoutError` and `InternalServerError`

- **File:** `folio/llm/providers.py`:165-180
- **Spec:** section 6.3
- **Found by:** R1, R2, R3 (unanimous)
- **Description:** The spec lists `APITimeoutError` and `InternalServerError` as transient for OpenAI. Neither is imported or checked. They fall through to `PERMANENT`. Also missing: explicit `PermissionDeniedError` -> `PERMANENT`.
- **Fix:** Import `APITimeoutError`, `InternalServerError` from `openai` and classify as `TRANSIENT`. Add `PermissionDeniedError` -> `PERMANENT`.

### B3: Google `classify_error` missing `InternalServerError` and `DeadlineExceeded`

- **File:** `folio/llm/providers.py`:234-242
- **Spec:** section 6.3
- **Found by:** R1, R2, R3 (unanimous)
- **Description:** The spec lists `DeadlineExceeded` and `InternalServerError` as transient for Google. The string-based `classify_error` only checks for `ResourceExhausted` and `ServiceUnavailable`.
- **Fix:** Add `"InternalServerError"` and `"DeadlineExceeded"` to the transient string checks.

## Should-Fix Issues (7)

### S1: Profile name regex validation not enforced

- **File:** `folio/config.py`:176-206
- **Spec:** section 3.2
- **Found by:** R1, R2, R3 (unanimous)
- **Description:** Spec requires profile keys to match `^[a-z][a-z0-9_]*$`. `_validate_llm()` does not enforce this. Names like `"123-bad"` or `"My Profile"` are silently accepted.
- **Fix:** Add `import re` and a regex check in `_validate_llm()` for each profile key.

### S2: No test coverage for `_llm_metadata` in frontmatter

- **File:** tests/ (missing)
- **Spec:** sections 7.4, 9.6
- **Found by:** R1, R2 (agreement)
- **Description:** Spec requires tests for `_llm_metadata` written to frontmatter. No test verifies the structure: `_llm_metadata.convert.{requested_profile, profile, provider, model, fallback_used, status, pass2.status}`.
- **Fix:** Add test calling `frontmatter.generate()` with `llm_metadata` and asserting the parsed YAML output.

### S3: Dead imports `base64` and `os` in analysis.py

- **File:** `folio/pipeline/analysis.py`:3,7
- **Found by:** R1, R2, R3 (unanimous)
- **Description:** After refactoring provider logic out of analysis.py, `import base64` and `import os` are unused.
- **Fix:** Remove both imports.

### S4: No timeout configured for Google Gemini API calls

- **File:** `folio/llm/providers.py`:210-217
- **Found by:** R3
- **Description:** Anthropic and OpenAI adapters set `timeout=120.0`. The Google adapter passes no timeout. A hung Google API call blocks indefinitely.
- **Fix:** Add timeout via `http_options` in `GenerateContentConfig` or at client construction.

### S5: `StageLLMMetadata.profile_name` is never populated

- **File:** `folio/llm/types.py`:105, `folio/pipeline/analysis.py`:355,764
- **Found by:** R2
- **Description:** `StageLLMMetadata.profile_name` is initialized to `""` and never set. The converter constructs `_llm_metadata` from `profile.name` directly, making this field dead data.
- **Fix:** Either populate it in `analyze_slides()`/`analyze_slides_deep()` or remove it from `StageLLMMetadata`.

### S6: Fallback chain exhaustion returns `primary_name` as `used_provider` (incorrect provenance)

- **File:** `folio/pipeline/analysis.py`:494-501
- **Found by:** R3
- **Description:** When all providers are exhausted, `_analyze_with_fallback` returns `primary_name` as `used_provider`. Cache stores `_provider=primary_name` even though fallback providers were attempted. Provenance is misleading.
- **Fix:** Return last-attempted provider or a sentinel like `"exhausted"`.

### S7: Deep cache entries missing per-slide `_provider`/`_model` provenance

- **File:** `folio/pipeline/analysis.py`:880-886
- **Spec:** section 7.3
- **Found by:** R3
- **Description:** Pass-1 cache entries store `_provider` and `_model` per slide (line 439-440). Pass-2 deep cache entries do not. When fallback activates during Pass 2, per-slide provenance is lost.
- **Fix:** Store `_provider`/`_model` in deep cache entries the same way pass-1 does.

## Spec Deviation Analysis

The following intentional deviations from the spec were identified. None are blocking; all are pragmatic improvements documented here for traceability.

| Deviation | Spec says | Implementation does | Assessment |
|-----------|-----------|---------------------|------------|
| `ErrorDisposition` shape | Dataclass with `kind`, `message`, `original` | Enum with `TRANSIENT`/`PERMANENT` | Simpler; orchestrator handles messages independently |
| `ProviderInput` fields | `image_bytes: bytes`, `media_type: str` | `image_path: Path` | Each adapter reads bytes; avoids unnecessary memory copy |
| `ProviderOutput` field name | `text: str` | `raw_text: str` + extra provenance fields | More descriptive; extra fields useful |
| `create_client` signature | `api_key: str` | `api_key_env: str` (env var name) | Adapter resolves key; cleaner for BYO credential wiring |
| Registry | Singleton instances | Classes instantiated per lookup | Stateless providers; no behavioral difference |
| Client lifetime | One per conversion invocation | One per pass (Pass 1 and Pass 2 each create a client) | Works but doubles client creation overhead |

## Positive Observations (cross-reviewer consensus)

- **Clean abstraction boundary:** Zero `import anthropic` in analysis.py. No provider-specific payload construction outside adapters. No `isinstance` checks for SDK types outside providers.py.
- **Spec-compliant cache design:** Invalidation on provider/model/prompt/extraction version change. Profile rename correctly excluded. Atomic writes via `.tmp` + `rename`.
- **Narrow `_extract_json`:** Direct parse, then one code-fence strip, then fail. No regex scraping. Matches spec section 5.8.
- **Correct fallback gate:** Only transient errors trigger fallback. Permanent errors, truncation, and malformed JSON never do.
- **Thorough JSON normalization:** All 6 rules from spec section 5.6 implemented (lowercase-hyphenation, element_type, confidence, cap at 10, zero-evidence rejection).
- **Config fail-fast:** Unsupported providers, missing `routing.default`, dangling route targets, dangling fallback references all caught at load time.
- **CLI override semantics:** `--llm-profile` correctly disables fallback chains per spec section 3.5.
- **Legacy config migration:** Synthetic `default_<provider>` profile with both `default` and `convert` routes per spec section 3.4.
