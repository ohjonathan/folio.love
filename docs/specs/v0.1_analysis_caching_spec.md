**Status:** Ready for implementation
**Type:** Hardening / completion (not new-build)
**Estimated effort:** 2-3 days
**Branch:** `feat/analysis-cache-hardening`
**Revision:** Rev 2 — addresses 1/3 review approvals; fixes B-1 (`--no-cache` semantics), B-2 (end-to-end cascade test), S-1 through S-4, m-1 through m-6

## 1. Overview

Per-slide analysis caching already exists and works correctly for the common case:

- **Pass 1 cache:** `.analysis_cache.json` keyed by SHA256 image hash (16 hex chars)
- **Pass 2 cache:** `.analysis_cache_deep.json` keyed by `{image_hash}_deep`
- **Prompt invalidation:** `_prompt_version()` hashes the first 500 chars of the prompt; cache is fully invalidated when the prompt changes
- **Atomic writes:** temp file + rename for all cache files
- **Result:** On reconversion with no source changes, 0 API calls (all cache hits)

This spec addresses **7 gaps** — 3 blocking cache correctness issues identified during review, plus 4 original observability/usability gaps:

### Blocking (from review)

| # | Gap | Risk | Severity |
|---|-----|------|----------|
| B1 | Pass 1 cache key omits text inputs | Same image + different speaker notes = stale cache hit | **Blocking** |
| B2 | Pass 2 cache key omits pass-1 context | Pass-1 change serves stale deep results | **Blocking** |
| B3 | Legacy markerless caches trusted unsafely | Unknown caches load without validation | **Blocking** |

### Original gaps

| # | Gap | Risk | Severity |
|---|-----|------|----------|
| G1 | Model version not in cache metadata | Switching models serves stale results | Medium |
| G2 | `_EXTRACTION_VERSION` not wired in | Text extraction logic changes don't invalidate cache | Medium |
| G3 | No `--no-cache` CLI flag | No way to force re-analysis | Low |
| G4 | No cache hit/miss stats | Cache behavior invisible to users | Low |

### B1: Pass 1 cache key omits text inputs

**Problem:** The cache key is `image_hash` only, but the API prompt includes `slide_text.full_text` + elements via `_build_text_context()` (analysis.py:106). Same image with different speaker notes produces a stale cache hit because the text difference isn't captured.

**Evidence:** `_analyze_single_slide()` at line 234 calls `_build_text_context(slide_text)` and appends the result to `ANALYSIS_PROMPT`. The cache lookup at line 203 checks only `image_hash in cache`. Elements are deterministically derived from `full_text` by `_detect_elements()`, so hashing `full_text` alone is sufficient.

### B2: Pass 2 cache key omits pass-1 context

**Problem:** The deep cache key is `{image_hash}_deep`, but `DEPTH_PROMPT` is built from pass-1 outputs (`slide_type`, `framework`, `key_data`, `main_insight`) at analysis.py:616-620. Text context is also appended at line 702. If pass-1 results change (e.g., due to a text change or model switch), the deep cache serves stale results.

**Cascade chain:** text change -> pass-1 miss -> new pass-1 results -> `_pass1_hash` changes -> pass-2 miss. The `_text_hash` in deep cache provides defense-in-depth for edge cases where pass-1 results happen to be identical despite different text.

### B3: Legacy markerless caches trusted unsafely

**Problem:** Analysis cache uses a lenient guard (`if stored is not None and stored != current`), so caches without markers load fine (analysis.py:812-815). Compare to the strict pattern in versions.py:222 (`if stored != current` — missing marker = invalidate). A correctness-hardening spec should not silently trust unknown caches.

**Resolution:** One-time full invalidation of all legacy caches on first post-upgrade run via `_ANALYSIS_CACHE_VERSION`. This has a key simplifying effect: since all v1 entries will have the required markers, B1/B2 entry-level checks don't need None/missing fallback paths, `_prompt_version` checks can become strict, and the old-format deep cache branch (`isinstance(cached, list)` at analysis.py:601) can be removed.

## 2. Scope

### In scope

- B1-B3 (blocking cache correctness) and G1-G4 (original gaps) as described above
- S1: Hash full prompt text (not just first 500 chars)
- S2: CacheStats on early returns (zeros)
- New test file `tests/test_analysis_cache.py` (~29 tests)
- Update existing test call sites for new return type (G4)

### Out of scope

- **Cache pruning** — cache files are small (~1KB/slide), not a correctness issue
- **Cross-deck cache sharing** — caches are per-deck-directory by design
- **`--clear-cache` flag** — not needed. `--no-cache` now overwrites stale entries with fresh results (see G3).

### 4.1 B3 + S1: Cache format version and full prompt hash (foundation)

**Problem:** Legacy caches without markers are trusted silently. `_prompt_version()` hashes only the first 500 chars of prompts that exceed 1,500 chars.

**Design:**

Add `_ANALYSIS_CACHE_VERSION = 1` constant. Use strict check matching the pattern in versions.py:222:

```python
_ANALYSIS_CACHE_VERSION = 1

def _load_cache(cache_dir: Path, model: str | None = None) -> dict:
    cache_file = cache_dir / ".analysis_cache.json"
    if cache_file.exists():
        try:
            data = json.loads(cache_file.read_text())
            if not isinstance(data, dict):
                logger.warning("Cache is not a dict — resetting")
                return {}
            # Strict format version check — invalidates all legacy caches
            if data.get("_cache_version") != _ANALYSIS_CACHE_VERSION:
                logger.info("Cache format version mismatch — invalidating")
                return {}
            # Strict prompt version check (no `is not None` guard)
            if data.get("_prompt_version") != _prompt_version(ANALYSIS_PROMPT):
                logger.info("Analysis prompt changed — invalidating cache")
                return {}
            # ... model and extraction version checks (G1, G2) ...
            return data
        except (json.JSONDecodeError, OSError):
            return {}
    return {}
```

Update `_prompt_version()` to hash the entire prompt (remove `[:500]`):

```python
def _prompt_version(prompt_text: str) -> str:
    """Hash a prompt to detect prompt changes."""
    return hashlib.sha256(prompt_text.encode()).hexdigest()[:8]
```

Update `_save_cache()` and `_save_cache_deep()` to write `_cache_version`, with OSError protection:

```python
def _save_cache(cache_dir: Path, cache: dict, model: str | None = None):
    try:
        cache["_cache_version"] = _ANALYSIS_CACHE_VERSION
        cache["_prompt_version"] = _prompt_version(ANALYSIS_PROMPT)
        # ... model and extraction version (G1, G2) ...
        tmp_file = cache_file.with_suffix(".tmp")
        tmp_file.write_text(json.dumps(cache, indent=2))
        tmp_file.rename(cache_file)
    except OSError as e:
        logger.warning("Failed to write cache %s: %s", cache_file, e)
```

Apply the same pattern to `_load_cache_deep()` / `_save_cache_deep()` (~5 lines each). Note: analysis results are already in memory and returned regardless — only the cache persistence fails gracefully.

**~30 lines changed across 4 functions.**

### 4.2 B1: Per-slide text hash in Pass 1 cache

**Problem:** Cache key is `image_hash` only, but the API prompt includes `slide_text.full_text`.

**Design:** Keep `image_hash` as the dict key. Store `_text_hash` (SHA256[:16] of `slide_text.full_text`) inside each cached entry. On lookup, compare stored vs current text hash — mismatch = cache miss.

**Why stored metadata over composite key:**
- No key format migration (existing entries still match by `image_hash`)
- Debuggable (can correlate cache entries to slide images)
- Handles Optional text gracefully (hash of `""` when no text)

**Why hash `full_text` only (not `elements`):** Elements are deterministically derived from `full_text` by `_detect_elements()`. If `full_text` changes, elements change.

**New helper:**

```python
def _text_hash(slide_text: Optional["SlideText"]) -> str:
    """Hash slide text for cache validation."""
    text = slide_text.full_text if slide_text and slide_text.full_text else ""
    return hashlib.sha256(text.encode()).hexdigest()[:16]
```

**Updated cache check in `analyze_slides()`:**

```python
if image_hash in cache:
    cached_entry = cache[image_hash]
    current_text_hash = _text_hash(slide_texts.get(i) if slide_texts else None)
    if cached_entry.get("_text_hash") != current_text_hash:
        logger.info("Slide %d: text changed — cache miss", i)
        # Fall through to API call
    else:
        logger.debug("Slide %d: using cached analysis", i)
        results[i] = SlideAnalysis.from_dict(cached_entry)
        stats.hits += 1
        continue
```

**Updated cache store:**

```python
if cache_dir:
    entry = analysis.to_dict()
    entry["_text_hash"] = _text_hash(slide_texts.get(i) if slide_texts else None)
    cache[image_hash] = entry
```

**No None/missing fallback needed:** B3 invalidates all legacy entries on first run, so all v1 entries will have `_text_hash`.

**`slide_texts=None` behavior:** When `slide_texts` is None or a slide has no text, `_text_hash()` hashes the empty string. This means a cache entry created without text will correctly miss when text becomes available (and vice versa).

### 4.3 B2: Pass-1 context hash + text hash in Pass 2 cache

**Problem:** Deep cache key is `{image_hash}_deep`, but `DEPTH_PROMPT` is built from pass-1 outputs and text context is appended.

**Design:** Store `_pass1_hash` (SHA256[:16] of pass-1 fields) and `_text_hash` in each deep cache entry. Both must match for a cache hit.

**New helper:**

```python
def _pass1_context_hash(analysis: SlideAnalysis) -> str:
    """Hash pass-1 fields that feed into the depth prompt."""
    content = f"{analysis.slide_type}|{analysis.framework}|{analysis.key_data}|{analysis.main_insight}"
    return hashlib.sha256(content.encode()).hexdigest()[:16]
```

> **Why evidence is excluded:** Evidence is not an input to `DEPTH_PROMPT` — only `slide_type`, `framework`, `key_data`, and `main_insight` are interpolated (analysis.py:616-620). If evidence changes while these four fields remain identical, the depth prompt is byte-for-byte the same, so a cache hit is correct. The `_text_hash` guard provides additional coverage since evidence changes typically correlate with text changes.

**Updated cache check in `analyze_slides_deep()`:**

```python
if deep_key in deep_cache:
    cached = deep_cache[deep_key]
    current_text_hash = _text_hash(slide_texts.get(slide_num))
    current_pass1_hash = _pass1_context_hash(results[slide_num])
    if (cached.get("_text_hash") != current_text_hash or
            cached.get("_pass1_hash") != current_pass1_hash):
        logger.info("Slide %d: inputs changed — deep cache miss", slide_num)
        # Fall through to API call
    else:
        logger.debug("Slide %d: using cached deep analysis", slide_num)
        new_evidence = cached.get("evidence", [])
        reassessed_type = cached.get("pass2_slide_type")
        reassessed_framework = cached.get("pass2_framework")
        stats.hits += 1
        # ... merge evidence ...
        continue
```

**Updated cache store:**

```python
if cache_dir:
    deep_cache[deep_key] = {
        "evidence": new_evidence,
        "pass2_slide_type": reassessed_type,
        "pass2_framework": reassessed_framework,
        "_text_hash": _text_hash(slide_texts.get(slide_num)),
        "_pass1_hash": _pass1_context_hash(results[slide_num]),
    }
```

**Remove old-format branch:** The `isinstance(cached, list)` branch at analysis.py:601-612 handles a pre-v1 cache format. Since B3 invalidates all legacy caches, this branch is dead code and should be removed.

### 4.6 G4: Cache hit/miss statistics

**Problem:** Cache hits are logged at DEBUG only. Users have no visibility into whether caching is working.

**Design:** Add a `CacheStats` dataclass and change `analyze_slides()` and `analyze_slides_deep()` to return `tuple[dict, CacheStats]`. Display stats at INFO level and in CLI output.

**CacheStats dataclass:**

```python
@dataclass
class CacheStats:
    """Cache hit/miss statistics for a single analysis pass."""
    hits: int = 0
    misses: int = 0
    pass_name: str = "pass1"

    @property
    def total(self) -> int:
        return self.hits + self.misses

    @property
    def hit_rate(self) -> float:
        return self.hits / self.total if self.total > 0 else 0.0

    def merge(self, other: "CacheStats") -> "CacheStats":
        """Merge stats from another pass (e.g., pass2 into pass1)."""
        return CacheStats(
            hits=self.hits + other.hits,
            misses=self.misses + other.misses,
            pass_name="combined",
        )
```

Known limitation: `merge()` loses per-pass breakdown (hard-codes `pass_name="combined"`). Per-pass stats are preserved in INFO logging.

**Return type changes:**

```python
def analyze_slides(...) -> tuple[dict[int, SlideAnalysis], CacheStats]:
    stats = CacheStats(pass_name="pass1")
    # ... hits/misses counted in loop ...
    logger.info("Pass 1 cache: %d hits, %d misses (%.0f%% hit rate)",
                stats.hits, stats.misses, stats.hit_rate * 100)
    return results, stats

def analyze_slides_deep(...) -> tuple[dict[int, SlideAnalysis], CacheStats]:
    stats = CacheStats(pass_name="pass2")
    # ... hits/misses counted in loop ...
    logger.info("Pass 2 cache: %d hits, %d misses (%.0f%% hit rate)",
                stats.hits, stats.misses, stats.hit_rate * 100)
    return results, stats
```

**S2: Early return semantics.** All early-return paths (no API key, import error, no dense slides) return `CacheStats()` with zeros. Consumer can check `stats.total == 0` to distinguish "no analysis happened" from "all hits."

```python

# Import error
except ImportError:

# No dense slides (analyze_slides_deep)

    return pass1_results, CacheStats(pass_name="pass2")
```

### 4.7 G3: `--no-cache` CLI flag

**Problem:** No way for users to force re-analysis. The original design set `cache_dir=None`, which skipped both read AND write — meaning a `--no-cache` run to escape bad analysis left the stale cache intact for the next normal run.

**Design:** Skip cache reads but still write fresh results. The converter always passes `cache_dir=deck_dir`. A new `force_miss: bool = False` parameter on `analyze_slides()` and `analyze_slides_deep()` controls whether cached entries are consulted. When `force_miss=True`, `_load_cache()` returns `{}` but `_save_cache()` still runs, overwriting stale entries with fresh analysis.

**Changes to `folio/pipeline/analysis.py`:**

```python
def analyze_slides(
    ..., cache_dir: Path | None = None,
    force_miss: bool = False,
) -> tuple[dict[int, SlideAnalysis], CacheStats]:
    ...
    cache = _load_cache(cache_dir, model=model) if cache_dir and not force_miss else {}
    # ... analysis loop ...
    if cache_dir:
        _save_cache(cache_dir, cache, model=model)  # always writes
```

Same pattern for `analyze_slides_deep()`.

**Changes to `folio/converter.py`:**

```python
def convert(
    self,
    source_path: Path,
    note: Optional[str] = None,
    client: Optional[str] = None,
    engagement: Optional[str] = None,
    target: Optional[Path] = None,
    passes: Optional[int] = None,
    no_cache: bool = False,
) -> ConversionResult:
    ...
    # Always pass cache_dir — force_miss controls read behavior
    slide_analyses, pass1_stats = analysis.analyze_slides(
        image_paths, model=self.config.llm.model,
        cache_dir=deck_dir, slide_texts=slide_texts,
        force_miss=no_cache,  # converter.py:117
    )
    ...
    slide_analyses, pass2_stats = analysis.analyze_slides_deep(
        ..., cache_dir=deck_dir,
        force_miss=no_cache,  # converter.py:133
    )
```

**Changes to `folio/cli.py`:**

```python

              help="Force re-analysis; fresh results replace cached entries.")
```

Add to both `convert` and `batch` commands.

## 7. Migration and Compatibility

**One-time full invalidation of legacy caches.**

The `_ANALYSIS_CACHE_VERSION = 1` strict check causes all pre-v1 caches (those without `_cache_version` or with a different version) to be invalidated on first post-upgrade run. This is intentional: **correctness over continuity.**

**What happens on first post-upgrade run:**

1. `_load_cache()` reads existing `.analysis_cache.json`
2. `data.get("_cache_version") != 1` -> returns `{}`
3. All slides are re-analyzed via API
4. `_save_cache()` writes v1 cache with all markers (`_cache_version`, `_prompt_version`, `_model_version`, `_extraction_version`, per-slide `_text_hash`)
5. Same process for deep cache

**Why this is acceptable:**
- First-run cost is bounded (one full re-analysis, same as a fresh conversion)
- Eliminates all legacy-format edge cases (no None/missing fallback paths needed)
- Enables removal of old-format deep cache branch (`isinstance(cached, list)` at analysis.py:601). This branch handled a pre-v1 format where deep cache stored raw evidence lists instead of dicts with metadata. B3 invalidation makes it dead code.
- All subsequent runs benefit from correct, comprehensive cache validation

**New cache files opened by old code:** Old code ignores unknown keys — no crash, no data loss. However, old code will not perform the new validation checks, so users should not downgrade after upgrading.
