### 1.3 Implementability

Could a third-party operator with no johnny-os context dispatch these and get useful results? **Partially.**

- **Token Substitution Fragility:** The "composition" model (Preamble + Contract + Role) requires the operator to perform three separate substitution passes or one massive one. Without a v2 generator, the likelihood of an `<ANGLE_UPPER>` string surviving into a prompt is high.
- **Undefined Jargon:** Templates reference "Doc Index Tool" (`ontos`) and "PR Tool" (`gh`) as tokens, but the instructions for workers on *how* to use them are underspecified. A worker in a fresh session won't know the CLI flags for a custom `<DOC_INDEX_TOOL>`.
- **Phase A Void:** An operator starting at Phase A must either write their own meta-prompt or use the johnny-os parent playbook, breaking the "portable" boundary of the bundle.

### 1.4 Cross-Artifact Coherence

- **Principle P3 Conflict:** `framework.md:96` says "The same model family may not hold more than one role on the same artifact in the same round." `playbook.md:97` suggests "Double-role one family with two sessions" as a valid option. The `walkthrough` (L75) then uses Codex as both Peer and Adversarial. This is a critical internal contradiction between Doctrine and Playbook.
- **Phase Naming:** `framework.md` uses phases `0-A-B-C-D-E`. Templates like `03` use `<PHASE_ID>`, but `tokens.md` (L38) says `<PHASE_ID>` is "Phase letter (0,A,B,C,D,E) optionally with sub-step (D.2)." However, the Orchestrator (`00:99`) uses `<PHASE_ID+1>` which is not a defined token.

### 1.5 Operator UX

The `walkthrough-mini-deliverable.md` is the most honest document in the bundle. It reveals that:
1. P3 is effectively impossible to hold with only 3 families (Claude, Codex, Gemini) when one family is the author.
2. The solution ("skip the least critical lens") is not codified in `framework.md`.
3. The operator must "manually check" (L32) the manifest against the schema, which is a significant friction point for a framework claiming "mechanical" discipline.

## Pass 3: Adversarial Review

**Lens:** Failure modes, edge cases, attacks, blind spots.
**Core question:** How does this break?
**Mindset:** I am actively trying to break this bundle. Suppression of helpfulness.

### 3.1 Assumption Attack

- **Assumption:** "Token substitution is mechanical."
- **Attack:** If the operator substitutes `<SMOKE_CHECKS>` as a multi-line block into the Orchestrator's markdown table (`00:105`), the table breaks. The template assumes scalar values in a context where only scalars work.
- **Assumption:** "One role per session (P10) prevents bias."
- **Attack:** If the same model *family* holds two roles in separate sessions (as recommended in `playbook.md:97`), the underlying bias is identical. P10 is a false security boundary against shared model weights.

### 3.2 Failure Mode Analysis

- **Scenario:** **The "Halt Loop".** A worker halts (P1), the orchestrator remediates and dispatches a continuation (11). The continuation worker halts on the same issue. There is no "Max 3 Halts" principle in `framework.md`. The system can enter an expensive infinite loop of resume-after-halt.
- **Scenario:** **The "Orchestrator Drift".** The Orchestrator runbook (`00`) defines the state machine, but the `02` (Dispatch) template allows the operator to add new halt conditions. If the operator adds a halt condition that contradicts `00`, the orchestrator session will hang or misroute.

### 3.3 Portability Attack (Beyond the Grep)

- **The `scripts/` Assumption:** `example-manifest.yaml:113` references `scripts/spec-coverage.sh`. The bundle does not include `scripts/`. A third-party operator will have a broken manifest gate (G2) immediately upon adoption.
- **The `gh` Dependency:** `tokens.md:28` examples `gh` for `<PR_TOOL>`. While a token, the `00` template assumes a PR tool exists that can be "polled" or "watched" (L76). This implicitly targets GitHub/GitLab.

### 3.4 Recursive Trap Analysis

**The "Consolidator Paradox":**
The Framework Doctrine (`framework.md`) defines the Meta-Consolidator role (`P5`) as preserving blockers with file:line evidence. 
**Attack:** If I (Gemini) provide a blocking finding in this review with file:line evidence, and another model family (Claude/Codex) approves the bundle, the framework's own `06` template would *force* the consolidator to preserve my blocker. 
**Trap:** The bundle's value is dependent on a Consolidator template (`06`) that it claims to define, but if the bundle is "Blocked" by this review, the very template (`06`) used to adjudicate the review is part of the blocked artifact. This is a recursive trust dependency.

### 3.5 Manifest / Generator Attack

- **Under-specification:** `generator-spec.md` does not define the "empty value rendering policy" for optional tokens like `<FORBIDDEN_SYMBOLS?>`. Does the generator emit an empty string, a placeholder, or skip the line? Two implementers *will* produce different results, breaking portability.
- **Schema Escape:** The schema allows `additionalProperties: false` at the root, but not inside `scope`. An operator can inject ad-hoc scope rules that the generator won't know how to substitute into the templates.

### 3.6 Risk Assessment Override

The Build Plan implies **LOW** risk for v1. **I DISAGREE.**
The risk is **MEDIUM-HIGH** because:
1. The doctrine (P3) is unfalsifiable when using only 3 models (the industry standard for this team).
2. The lack of a Phase A template means the framework is "broken" out of the box for its first creative step.
3. Token substitution is underspecified to the point of operational failure.

### 3.10 Adversarial Issues Found

| ID | Severity | Location | Attack | Reproduction | Fix |
|----|----------|--------|--------|--------------|-----|
| X-1 | **Critical** | `templates/01` | **Token Substitution Injection.** | Substitute a multi-line script into `<SMOKE_CHECKS>` inside the `01` contract block. | Define clear "rendering policies" for multi-line tokens in `generator-spec.md`. |
| X-2 | **Major** | `framework.md` | **Infinite Halt Loop.** | A worker halts on a capability-mismatch; orchestrator re-dispatches to the same model. | Add a "Circuit Breaker" principle to Doctrine: halt the entire deliverable after 2 consecutive halts on the same step. |
| X-3 | **Major** | `generator-spec.md` | **Implementation Divergence.** | Two developers build a v2 generator; one strips optional tokens, one leaves `<TOKEN?>`. | Specify exact string-replacement behavior for optional tokens. |

### 3.11 Adversarial Verdict

**BLOCK.** The bundle is not ready for v1.0 stamp. The recursive trust issues, the manifest-schema mismatches, and the critical "Token Substitution Injection" risk mean this framework will break when exposed to real-world multi-line shell scripts or ad-hoc operator choices.
