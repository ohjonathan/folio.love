### ALIGN-BLOCKER1 - Same-provider adversarial escape hatch accepts a dummy gate id

- **Raised by:** codex / Alignment
- **Description:** A v1.2 manifest can satisfy the same-provider
  adversarial escape hatch by adding a bare `gate_prerequisites[].id` equal
  to `G-xprov-adv-B.1`. That lets an author-family `claude-opus` manifest
  use same-provider `claude-sonnet` Adversarial at B.1 and still pass
  adopter verification, without documenting or proving the cross-provider
  second-pass artifact the framework requires.
- **Location:** `framework.md:164-174`; `scripts/verify-p3.sh:170-188`
- **Evidence:** direct-run
- **Reproduction:** Alignment synthesized
  `/tmp/llm-dev-v1.2-same-provider-dummy-escape-exact.yaml` by changing the
  v1.0 example to `manifest_version: "1.2.0"` and inserting a prerequisite
  with `id: G-xprov-adv-B.1`, `category: test`, and `command: "true"`.
  `bash frameworks/llm-dev-v1/scripts/verify-adopter.sh
  /tmp/llm-dev-v1.2-same-provider-dummy-escape-exact.yaml` returned
  `PASSED (4/4 checks)`.
- **Required action:** Make the escape hatch structured and checkable, or
  remove it. A sufficient fix would record cross-provider adversarial pass
  data with phase, family, provider, and artifact path; validate that the
  pass provider differs from the author provider; and require a gate tied to
  that artifact before D.6. At minimum, `verify-p3.sh` must reject a bare
  id-only/no-op prerequisite.
