### Second continuation: subtraction-operator gap (2026-04-15)

A follow-up review noted that the pseudo-token scanner still allowed
`<PHASE_A-PHASE_B>` because the operator character class was `[+*/]`.
Fix: extended the class to `[+*/-]` (hyphen placed last so it is treated
as literal, not a range).

Regression-tested: injecting `<PHASE_A-PHASE_B>` into any template
triggers a direct-run failure with file:line. The earlier three
operator forms (`+`, `*`, `/`) remain caught.
