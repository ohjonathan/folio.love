# Adversarial Review — frameworks-llm-dev-v1 / v1.2.0 / gemini

## 1. Assumption attack

| Assumption | Why it might be wrong | Impact if wrong |
|------------|------------------------|-----------------|
| "`verify-p3.sh` guarantees mechanical presence of the cross-provider artifact." | It checks if the `artifact_path` is a substring of `verification.command`. A vacuous entry like `command: "foo.md"` satisfies the static check without actually validating the file's presence at runtime. | Minor bypass: the orchestrator passes static checks but the D.6 runtime execution might just execute the markdown file (which fails) or do nothing if the command is `echo foo.md`. |
| "Families without `evidence_cap` safely default to `direct-run`." | A family might be completely sandboxed (e.g., `shell: false`, `test_runner: false`). Defaulting to `direct-run` grants them blocker-grade authority they physically cannot exercise, making the default incoherent. | A sandboxed family claims a `direct-run` blocker, and Template 06 does not auto-downgrade it because the implicit cap is `direct-run`. |

## 2. Failure mode analysis

| Failure | How it happens | Would we notice? |
|---------|----------------|-------------------|
| Orchestrator preflight crash | The new B.1/D.2 dispatch preflight in Template 02 uses `yq`. If the orchestrator machine does not have `yq` installed (it was not a declared dependency in v1.0/v1.1), the preflight halts. | Yes, the orchestrator experiences a command-not-found error and the dispatch queue validation fails. |

## 3. Diagram completeness attack

The framework correctly isolates the orchestrator fast-path to `manifest_version >= 1.2.0`. Older manifests retain the mandatory meta-consolidator step, preserving legacy compatibility. 

## 4. Edge case inventory

- **D.6 Gate malformed tables:** `verify-d6-gate.sh` is exceptionally strict. It rejects permuted columns, missing evidence classes, and disallowed evidence with exit 1.
- **Circuit breaker schema:** Duplicate rounds, missing `preserved_blocker_ids`, and invalid phase IDs are robustly caught by `check-jsonschema` or `verify-circuit-breaker.sh`.
- **Adopter space paths:** `verify-adopter.sh` cleanly handles spaces in manifest paths and symlinks.
- **day-one.sh onboarding:** Handles empty slugs, invalid formats (uppercase/underscores), and prevents overwriting existing manifests safely.

## 5. Security surface

- The `cross_provider_adversarial_passes` escape hatch is now fully structured and requires a different provider. The v1.1.0 ALIGN-BLOCKER1 bypass is closed.

## 6. Blind-spot identification

The default `evidence_cap` logic assumes all reviewer families are fully capable unless explicitly capped. For a security framework, defaulting to maximum authority (`direct-run`) rather than least privilege (`static-inspection`) is a conceptual blind spot, particularly for sandboxed families.

## 7. Risk-assessment override

The risk is **Low**. The mechanical gates built in v1.2 (`verify-d6-gate.sh`, `verify-circuit-breaker.sh`, `verify-adopter.sh`) are highly robust against adversarial inputs. The remaining gaps are tooling documentation (`yq`) and default configurations.
