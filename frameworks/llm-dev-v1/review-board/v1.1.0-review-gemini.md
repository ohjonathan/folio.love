Each blocker preserved exactly as raised. No silent demotion. Three

### GEMINI-B1 — Template 19 §2 temporal paradox (Peer)
- **Description:** Template 19 §2 "Product-surface cross-reference"
  required spec-vs-implementation matching, but Template 19
  dispatches in B.1/B.2 (Spec Review) where no implementation exists.
  An agent at Phase B cannot complete §2 without hallucinating an
  implementation or halting.
- **Location:** `templates/19-review-board-product.md:109-122`
- **Evidence:** `static-inspection`
- **Status:** **closed Batch D (`e9fbef7`)** — §2 restructured
  phase-aware: §2.1 spec-declared surfaces (all phases), §2.2
  spec-vs-implementation (D.2 only; marked "n/a (Phase B)" at B).
  Halt catalog extended.

### GEM-B1 — verify-artifact-paths.sh hardcoded to single manifest (Adversarial)
- **Description:** Script targets only `example-manifest.yaml`; never
  validates `product_verdict` placeholder shape for the user-facing
  manifest. A malformed product_verdict path passes all v1.1 checks.
- **Location:** `scripts/verify-artifact-paths.sh:22`
- **Evidence:** `direct-run`
- **Status:** **closed Batch B (`77c3fea`)** prior to Gemini's review
  (Gemini reviewed pre-fix state). Script now iterates
  `manifests=(…)` array covering both example manifests; product_verdict
  grammar enforced when present.

### GEM-B2 — artifacts.product_verdict schema-optional but practically required (Adversarial)
- **Description:** Schema marks product_verdict optional; downstream
  consumers running schema-only validation get false green.
  Verify-p3.sh enforces presence but schema doesn't.
- **Location:** `manifest/deliverable-manifest.schema.yaml:204-206`
- **Evidence:** `direct-run` — `check-jsonschema` on user-facing
  manifest with product_verdict deleted returns rc=0 prior to fix.
- **Status:** **closed Batch D (`e9fbef7`)** — schema gains top-level
  allOf/if/then enforcing product_verdict required when
  user_facing: true. Defense-in-depth alongside verify-p3.sh.
  `check-jsonschema` now returns rc=1 with "'product_verdict' is a
  required property" on the negative fixture; v1.0.0 example still
  validates unchanged (backward-compat preserved).
