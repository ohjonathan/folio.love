Update README and the handoff introduction, and do not claim the cross-family canonical verdict/session archive until they exist.

### CODEX-SF2 - Scope/non-goal wording hides patch-level contract changes
- **Raised by:** Alignment (ALIGN-SF2); Peer (CODEX-PEER-MIN1 as scan-compat supporting signal)
- **Evidence:** `static-inspection`
- **Location:** `ROADMAP.md:89-105`; `CHANGELOG.md:6-28`; `docs/handoffs/2026-04-15_folio-love-llm-dev-adoption.md:217-220`; `manifest/deliverable-manifest.schema.yaml:145-152`; `templates/08-retrospective.md:116-123`; `templates/06-meta-consolidator.md:171-186`
- **Description:** The release prose says "no schema or template-contract changes" while the patch intentionally widens the manifest `id` pattern, adds normative archive ordering to Template 08, adds normative B.3 cardinality re-baseline text to Template 06, and changes missing-validator behavior in `verify-schema.sh`. The changes are patch-safe, but the prose is too broad.
- **Suggested action:** Reword to "no breaking schema changes, no schema rename, no new templates, no token removals; existing templates receive patch-level normative clarifications." Add a labeled `Non-goals held:` line in CHANGELOG so the non-goal claim is scan-compatible with ROADMAP.

### CODEX-SF3 - Orchestrator-only token extraction scans explanatory prose
- **Raised by:** Adversarial (ADV-SF1); Peer (CODEX-PEER-MIN2)
- **Evidence:** `direct-run`
- **Location:** `scripts/verify-tokens.sh:51,56`; `tokens.md:204-232`
- **Description:** The C-2 extraction range scans the full `## Orchestrator-only tokens` section and captures the prose placeholder `<ANGLE_UPPER>` in addition to the six published orchestrator-only tokens. Baseline `verify-tokens.sh` still passes, but the extractor is broader than the documented data contract.
- **Suggested action:** Restrict extraction to the published bullet/list rows, or parse the section structure so prose examples cannot become suppressed token names.

### CODEX-SF4 - Missing orchestrator-only token section fails silently
- **Raised by:** Adversarial (ADV-SF2)
- **Evidence:** `direct-run`
- **Location:** `scripts/verify-tokens.sh:8,51`
- **Description:** In a copied bundle with the orchestrator-only section deleted, `verify-tokens.sh` exits 1 with zero diagnostic bytes because `set -euo pipefail` lets the empty awk/grep pipeline terminate the script before controlled reporting.
- **Suggested action:** Wrap the optional extraction pipeline so empty output is handled explicitly. If the section is mandatory, fail with a named diagnostic such as `verify-tokens: FAILED - Orchestrator-only tokens section missing`.
