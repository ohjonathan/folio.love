### ADV-SF1 — `verify-tokens.sh` silently aborts when orchestrator-only heading drifts
- **Lens:** Adversarial | **Evidence:** direct-run
- **Location:** `scripts/verify-tokens.sh:51-57` (awk + grep pipeline) + comment at `49-50`
- **Description:** `set -euo pipefail` + empty awk output + `grep -oE` on empty stdin produces `rc=1` with empty stdout and empty stderr when the `## Orchestrator-only tokens` heading is renamed (case, hyphen, or spacing drift). Diagnostically invisible to an operator running the script standalone; `verify-all.sh` saves face by naming `verify-tokens.sh` in the FAILED line, but the per-script stanza is blank.
- **Reproduction:** `cp frameworks/llm-dev-v1/tokens.md /tmp/tokens.md && sed -i '' 's/## Orchestrator-only tokens/## Orchestrator only tokens/' /tmp/tokens.md`; run a replica that points `tokens=/tmp/tokens.md`: `rc=1`, no output. Also confirmed by briefly swapping the canonical tokens.md for the renamed variant and running `verify-all.sh` — per-script stanza is blank, FAILED list names `verify-tokens.sh`.
- **Suggested action:** Add a guard: `if [[ ! -s "$tmp/orchestrator_only" ]]; then echo "verify-tokens: no orchestrator-only section detected in $tokens" >&2; fi` before the `comm` step, OR pipe `grep -oE … \|\| true` so pipefail doesn't kill the script silently, OR add an explicit check that the section heading matches before the awk runs.
