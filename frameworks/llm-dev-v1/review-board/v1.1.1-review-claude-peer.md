**C-1 regex scope lock.** The v1.1.1 regex
`^[a-z][a-z0-9]*([.-][a-z0-9]+)*$` is a proper superset of the v1.1.0
regex. I mentally tested the commit's stated accept/reject cases:
accepts `my-thing-v0.6.4` (matches `[a-z][a-z0-9]*([.-][a-z0-9]+)*` as
`my` + `-thing` + `-v0` + `.6` + `.4`); rejects `x..y` (would need
consecutive `..` — the group `([.-][a-z0-9]+)` requires the separator
followed by at least one `[a-z0-9]`), rejects `x.` (trailing dot — the
trailing `([.-][a-z0-9]+)*` only matches if followed by
`[a-z0-9]+`), rejects `.x` (no leading separator — first class is
`[a-z]`). Slug stays `^[a-z][a-z0-9-]*$` (dot-free). The schema
description at lines 146-150 matches the accept/reject cases the commit
message cites. The backward-compat guarantee (v1.0.0 manifests still
validate) holds because any legal v1.0.0 id (dash-only component
separators) is still matched by the v1.1.1 pattern.

**C-5 hard-fail ergonomics.** The verify-schema diagnostic messages
(lines 24-27) explicitly label the change ("v1.1.1") and cite the
rationale ("silent-skip was removed because an absent validator looks
identical to a passing one"). README marks the dependency required and
notes the exception in both the Prerequisites section and the Verification
"Install dependencies" footnote. Defense-in-depth prose; one reader who
only reads the scripts still gets the full story.

## 6. Positive observations

- **Commit atomicity is exemplary.** Each of the 5 scope commits
  (C-1..C-5, B-5) touches only the files that scope item's declaration
  names, with no spillover. Stamps and handoff are each their own commit.
  A reader doing `git log -p` can audit each fix independently — the
  kind of patch-release hygiene the v1.2 build plan should codify as
  standard practice for cherry-pick runs.
- **verify-tokens.sh awk flag-based range (C-2).** The script's comment
  at line 49-50 explicitly documents the ADV-SF-001 lesson from the
  v1.1.0 review: "an earlier range `/,/^## [^O]/` broke on sections
  starting with 'O'". Adopting a flag-based range is the right
  structural fix — it survives any future heading-letter mix. Good
  memory of prior-review feedback baked into the implementation.
- **C-5 prose defense-in-depth.** The hard-fail change is documented in
  three places (script diagnostic, verify-all.sh header comment, README
  Prerequisites + Install sections), each with the same rationale. A
  reader who only reads one file still gets the story. This is the
  right shape for a behavior change that affects exit codes.
- **Handoff doc forward-looking section.** The "v1.1.1 sync note" at
  `docs/handoffs/2026-04-15_folio-love-llm-dev-adoption.md:205-234`
  correctly anticipates the bidirectional cherry-pick flow (folio
  ships first, canonical syncs, folio re-syncs). The "do NOT patch the
  bundle inside folio.love without upstreaming" guardrail (line 171) is
  intact — the new sync-note section does not undermine the "canonical is
  johnny-os" direction, it just documents the already-happened exception
  where folio shipped the fixes first.
- **CHANGELOG "Source:" line.** The inline provenance
  (`folio.love#51 ab4bc2735660`) on the CHANGELOG v1.1.1 entry is the
  exact level of traceability an external auditor would want: a reader
  can `git show ab4bc273` downstream to confirm the same 5 fixes
  landed byte-equivalent.
- **ROADMAP "Non-goals held" symmetry.** The v1.1.1 Non-goals paragraph
  explicitly enumerates the backward-compat guarantees (v1.0.0 and
  v1.1.0 manifests validate under v1.1.1; no template introduced or
  removed; 8/8 green suite). This is exactly what a patch-release's
  Non-goals section should assert — precise, mechanical, and
  falsifiable by running verify-all.

## 7. Verdict

**Approve.**

Zero blockers. Two should-fix items (PEER-SF1 ROADMAP exit-criteria
forward-looking prose; PEER-SF2 PR# placeholder when #7 is already
known) are editorial polish that can land pre-merge in a single prose
touch-up, or defer to the post-merge stamp pass without blocking
bundle conformance. Four minor items are documentation / future-version
carryovers (template-level version hygiene, awk self-extraction edge,
verify-all.sh `set -u` symmetry, grammar clarification for example fills).

The scope is tight: 5 cherry-picks, correctly atomized, each with
clean round-trip cross-references. The verify suite is 8/8 green on
direct-run. The backward-compat guarantee is mechanically witnessed
(both example manifests still validate). The bundle is ready to merge.
