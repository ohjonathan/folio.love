## 4.6 Parallel-vs-sequential empirical diff

The baseline at `_baselines/v1.0.0-review-claude-sequential-baseline.md` is a prior Claude single-session run where Peer, Alignment, Adversarial, and Consolidator were held sequentially in one session — explicitly violating the bundle's P10 rule. This parallel run spawns three isolated subagents, one role each — P10-faithful.

**Both runs converge on the unanimous blockers.** The three "every lens agreed" findings (B-1 schema enum, B-2 example P3 violation, B-3 author/verifier template gap) appear in both modes with the same reproductions. That's a stability signal.

**The parallel run finds BLOCKS the sequential run missed:**
- **B-5** (`<AUTHOR_RISK_LEVEL>` empty-substitution broken English) — sequential had this as Should-fix X-8; parallel Adversarial promoted to Block with reproduction.
- **B-6** (undefined "round" → P3-vs-playbook contradiction) — sequential did not find this at all. The parallel Adversarial session traced the single use of "round" and the playbook's double-roling sanction to a direct contradiction. Highest-value lens-unique finding.
- **B-7** (canonical-vs-family verdict path derivation underspec) — sequential missed this entirely. The parallel Adversarial session enumerated three plausible implementation rules.
- **B-8** (P3 mandates ≥2 families vs README portability claim) — sequential missed this entirely.

**The sequential run finds a recursive finding the parallel run did NOT explicitly surface:**
- **P10 recursive violation** — the sequential run's Adversarial X-3 flagged that any one-session review board violates the bundle's own P10 rule. This parallel run did NOT restate that as a block, because each of the three subagents holds exactly one role; P10 is satisfied by execution. The finding becomes unfalsifiable from inside any single-lens session — it is only visible from a meta-layer that observes the session composition.

**Net:** parallel ran at higher cross-artifact sensitivity (found 4 cross-file contradictions sequential missed) and lower recursive-self-observation sensitivity (lost the P10 self-observation). This is the expected trade — isolation buys independence at the cost of whole-system visibility.

**Implication for the bundle:** the bundle's own P10 rule, followed strictly, systematically narrows adversarial coverage for recursive-self findings. If v1.0.0 had been stamped on the sequential run alone, it would have missed B-5/B-6/B-7/B-8 (4 Blocks). If v1.0.0 is stamped on the parallel run alone, it misses the P10 recursive self-observation. Both modes together catch more than either. The framework should acknowledge this — possibly by prescribing **one of each mode** for framework-self-reviews specifically, with the second-mode's finding surfaced as "framework recursion report."

**What this says about v1.0.0 stamp readiness:** Either mode alone finds at least 4 Blocks. Both modes agree the bundle is not stamp-ready. The empirical question "does the P10 rule produce better or worse review coverage?" is answered: it depends on target class. For cross-artifact contradictions, one-role-per-session is better. For recursive-self meta-observation, single-session multi-role is the only mode that catches them. A framework reviewing itself needs both.
