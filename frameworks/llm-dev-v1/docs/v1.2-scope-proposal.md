## 3. Circuit-breaker override classification

**Verdict: framework-too-strict.**

Evidence chain:

1. B.3-r1 verdict (`f9bae38`) preserved 3 blockers → A-r2 (`15666c5`) closed **all 3**.
2. B.3-r2 raised 3 **new** blockers in the improved v1.1 spec: child `runtime_asset_id` collision across workspaces, YAML cyclic-alias crash, integer overflow on `heartbeat_seconds` / `timeout_ms`. These are not the r1 blockers cycling — they are new findings against an improved artifact.
3. The circuit breaker fired on round count alone. HALT commit `048f4bd` body states: "Framework circuit breaker mandates halt after round 2."
4. User-authorized override → A-r3 (`ebee226`) closed all 3 r2 blockers plus 7 should-fixes.
5. B.3-r3 canonical verdict (`0b9e350`) was **Approve**. Three new round-3 claimed blockers were P5-downgraded as `static-inspection` only.
6. D.3 subsequently caught three real code-layer crashes (`RecursionError`, oversize-integer `ValueError`, `UnicodeDecodeError`) with direct-run reproductions. All three would have crashed downstream callers on malformed but otherwise parseable inputs.

**Interpretation.** The breaker correctly halts when the **same**
blockers cycle — that is real stagnation. It mis-halts when
adversarial review surfaces **new** blockers in a progressively
improved artifact, which is real convergence. Without the override,
a converging review that later caught 3 production-grade crash bugs
would have been killed.

**Implication for scope.** This is a v1.1 template bug, not a
user-override-correct-gate case. Split across v1.1.1 (prose) and
v1.2 (mechanical) per finding #4.

## 6. Non-findings — what the framework got right

Listed explicitly so v1.2 does not fix what is not broken.

- **Multi-family adversarial B.1 board.** Caught 3 real spec-level defects (child-ID collision, cyclic aliases, integer overflow) that single-lens review would have shipped. Keep the 3-family schedule unchanged.
- **P5 evidence rule.** Carried 6 of 13 claimed-blocker adjudications silently across B.3-r3 and D.3. Prevented unnecessary A-r4+. Works as designed.
- **D.3 → D.4 → D.5 fix loop.** Found and closed 3 production-grade loader crashes (`RecursionError`, huge-integer `ValueError`, invalid-UTF-8 `UnicodeDecodeError`) with clean regression tests. Zero deviations or scope-lock violations in D.4.
- **D.6 14-row mechanical gate.** Every row has direct-run evidence with an explicit reproduction command. No subjective judgment in merge authorization. Keep the checklist.
- **Meta-consolidator arithmetic rule ("consensus is evidence, not arithmetic").** D.3 had Claude+Gemini Approve vs Codex Needs-Fixes with 3 reproducible bugs; a purely-voting gate would have shipped the 3 loader crashes. The current rule is correct.
- **Manifest/schema ergonomics.** Schema validation was signal-dense. No false positives across the run. D.6 gate's section-count and scope-lock checks ran clean.
- **Worker-session contract.** No context-window pressure observed in any of 27 sessions across 3 families.
